## 줄기 1: 가격이 정당한가 → 시세 분석가

```
혼자서도 인생 최대 거래를 후회 없이 결정할 수 있다
    ↓ 그러려면?
가격이 정당한지 검증해야 한다 (실거래 P50 vs 호가)
    ↓ 왜 지금은 안 되나?
부동산 중개인이 보여주는 호가는 팔고 싶은 가격이다.
실제 거래가 이뤄진 P50(중앙값)을 모르면
호가가 비싼지 싼지 판단할 수 없다.
호갱노노·네이버 부동산 호가 데이터는 거래 완료가 아닌 마케팅 가격.
    ↓ 그래서 필요한 것은?
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
[기능] 시세 분석가
 - 국토부 실거래 API 동일 단지·평형 6개월 분포
 - P50 vs 호가 편차 % + 적정/고평가/저평가 라벨
 - 별점: 💰 시세 (1~5점) + "오를 가능성 ↓" 같은 한 마디 코멘트
 - 모든 수치에 출처 자동 표기
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

| 판단 | 결론 |
|---|---|
| 이 기능 없으면? | 검증의 핵심이 빠짐 — 시스템 가치 없음 |
| 중요도 | **필수** |

```

    ↓ 왜 지금은 안 되나?

도보 시간·실제 노선·학교 배정·주변 인프라를 직접 확인 못 한다.
지도만 보면 동네 분위기와 동선 적합성을 알 수 없다.
    ↓ 그래서 필요한 것은?

 - 데이터 소스: Kakao Local + 학군 정보 API (Phase 3)
```

## 줄기 3: 숨겨진 리스크는 무엇인가 → 리스크 분석가

```
혼자서도 인생 최대 거래를 후회 없이 결정할 수 있다
    ↓ 그러려면?
단지·거시 차원의 숨겨진 리스크를 알아야 한다
    ↓ 왜 지금은 안 되나?
지금 호가·입지가 좋아 보여도
- 단지: 노후도, 대수선/재건축 이력, 분쟁
- 거시: 금리 인상, DSR 강화, 입주 물량
같은 구조적 리스크는 일반인이 못 본다.
중개인은 거래 성사가 목적이라 리스크를 "다만~" 수준으로만 언급.
    ↓ 그래서 필요한 것은?
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
[기능] 리스크 분석가
 - 단지 리스크 1개 + 거시 리스크 1개 이상 의무 제시
 - 데이터 소스: 단지 정보(노후도·대수선), 한국은행 금리,
   금감원 DSR 가이드, 한국부동산원 입주 물량
 - 별점: ⚠️ 리스크 (1~5점, 낮을수록 위험) + "주의 필요"
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

| 판단 | 결론 |
|---|---|
| 이 기능 없으면? | "괜찮아 보이는데"로 결정 → 5년 뒤 후회 |
| 중요도 | **필수** — Second Opinion 차별점의 핵심 |

## 줄기 4: 내가 감당 가능한가 → 재무 분석가

```
혼자서도 인생 최대 거래를 후회 없이 결정할 수 있다
    ↓ 그러려면?
내 자산·소득으로 이 매물이 감당 가능한지 검증
    ↓ 왜 지금은 안 되나?
DSR 40% 규제, LTV(생애최초 80%·일반 70%), 정책대출(디딤돌·보금자리)
모두 내 소득·무주택 여부에 따라 한도가 달라지는데
이걸 본인이 직접 계산하기 어렵다. 중개인은 일반론만.
    ↓ 그래서 필요한 것은?
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
[기능] 재무 분석가
 - LTV·DSR 계산 + 일반 주담대 + 정책대출 매칭
 - 자기자본 + 대출 = 총 매수 가능 예산
 - 월 원리금 + 취득세·중개료 포함 총취득비용
 - 별점: 💳 재무 (1~5점) + "감당 가능 수준"
 - 모든 수치에 출처 명시
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

| 판단 | 결론 |
|---|---|
| 이 기능 없으면? | 근거 없는 예산 계획 → 무리한 대출 위험 |
| 중요도 | **필수** |

## 줄기 5: 5~10년 후 가치가 유지·상승하는가 → 미래가치 분석가

```
혼자서도 인생 최대 거래를 후회 없이 결정할 수 있다
    ↓ 그러려면?
5~10년 후 가치가 유지·상승하는지 검증 (실거주여도 자산이다)
    ↓ 왜 지금은 안 되나?
중개인은 "여기 호재 많아요"라고만 한다 (역세권 신설, 재건축, 학교).
호재의 출처·실현 가능성·일정은 따로 검증해야 진실인지 안다.
악재(인구 감소, 공급 과다)는 더더욱 말 안 한다.
    ↓ 그래서 필요한 것은?
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
[기능] 미래가치 분석가
 - 개발 호재 출처별로 정리 (서울시 도시계획, 국토부 발표 등)
 - 호재의 실현 가능성·일정 + 악재(공급, 인구) 함께
 - 별점: 🎯 미래가치 (1~5점) + "호재 검증 필요"
 - 단순 기대치(호재 있다 → 좋다) 금지
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

| 판단 | 결론 |
|---|---|
| 이 기능 없으면? | "지금 좋다"만 보고 5년 후 가치 하락 직격 위험 |
| 중요도 | **필수** |

## 보조 기능의 Why Tree 연결

### MC 인터뷰어는 어디에 연결되나?

```
핵심 가치: 후회 없이 결정한다
    ↓
5명 분석가가 맞춤 검증을 하려면 → 먼저 내 상황을 알아야 한다
    ↓ 왜 지금은 안 되나?
긴 폼 입력은 사용자가 압박을 느낀다. 부동산 첫 구매라 뭘 적어야 할지도 모른다.
대화하듯 5~6개만 짧게 묻는 게 자연스러운 정확성을 만든다.
    ↓
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
[기능] MC 인터뷰어
 - 5~6개 짧은 질문 (보유 자산, 대출 한도 총액, 회사 위치,
   출퇴근 수단, 우선순위 1~2개)
 - 답변은 부담 없는 범위·키워드 수준
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### 매물 주소 입력은 어디에 연결되나?

```
핵심 가치: 후회 없이 결정한다
    ↓
검증 대상 매물이 정확히 무엇인지 시스템과 사용자가 합의해야 한다
    ↓ 왜 지금은 안 되나?
"광명시 OO아파트 105동 1203호"를 자유 입력 받으면
오타·오인이 분석 결과를 왜곡한다. API 호출용 표준 주소로 변환 필요.
    ↓
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
[기능] 매물 주소 입력 (Scene 03)
 - Step 1: 주소 변환 (행정동/도로명 → API 표준)
 - Step 2: 지도에 핀 표시
 - Step 3: 사용자 컨펌 ("이 매물이 맞나요?")
 - 데이터 소스: Kakao Local API
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### 서기(종합 리포터)는 어디에 연결되나?

```
핵심 가치: 후회 없이 결정한다
    ↓
5명 분석가의 의견을 한 화면에서 비교·종합할 수 있어야 결정 가능
    ↓ 왜 지금은 안 되나?
5명이 각자 길게 말하면 사용자가 종합 못 한다.
"5명 중 3명이 신중 검토 권합니다" 같은 한 줄 합의가 없으면 길을 잃는다.
    ↓
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
[기능] 서기 (Clerk) — 종합 리포터
 - 5인 카테고리 별점 (1~5점) 종합
 - 종합 평점 (예: 3.2/5) + 합의 결론 한 줄
 - 핵심 쟁점 한 줄 (예: "입지 vs. 미래가치")
 - 후속 액션 3가지(드릴다운/대화/PDF) 안내
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## 결론: 5인 분석가 + 한 화면 종합이 차별점이다

기능을 추가하고 싶을 때의 질문:

> "이 기능에서 위로 올라갔을 때, 다섯 줄기 중 하나를 거쳐 핵심 가치('후회 없이 결정')에 도달하는가?"

도달하면 만든다. 도달하지 않으면 만들지 않는다.

> "이 기능이 종합 리포트 한 화면에 어떤 형태로 보이는가?"

그릴 수 없으면 다시 생각.
