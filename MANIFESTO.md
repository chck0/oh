# Data 기반 Multi-Agent 부동산 자문 시스템 Manifesto

> 우리가 왜 이걸 만드는지, 어떤 기준으로 결정하는지에 대한 약속

---

## 우리가 이걸 만드는 이유 (Why We Exist)

한국에서 생애 첫 집을 사는 일은, 인생에서 가장 큰 돈을 쓰는 결정이다.

그래서 사람들은 멈춰있다. 정보가 없어서가 아니다. 확신이 없어서다.

확신이 없으면 행동할 수 없다. 행동하지 못하면, 집이 주는 안정감도, 인생의 다음 막도 시작되지 않는다.

우리 시스템은 정보를 주는 게 목적이 아니다. 여러 관점의 토론을 통해 확신을 만들어주는 것, 그래서 멈춰있는 사람을 움직이게 하는 것 — 그게 우리가 존재하는 이유다.

---

## 한 줄 정의

**생애 첫 집을 사려는 30대가, 여러 관점을 가진 AI 페르소나들의 토론을 통해 확신을 갖고 행동할 수 있도록 돕는 자문 시스템**

---

## 타겟 사용자

**생애 첫 주택 구매를 앞둔 30대 직장인**

- 일정한 소득이 있고, 어느 정도 모아둔 자금이 있으며, 신용도가 좋아 대출은 가능한 사람
- 하지만 부동산을 한 번도 사본 적 없어서, **무엇을 모르는지조차 모르는** 상태
- 공인중개사를 덜컥 만나기엔 무섭고, 유튜브를 봐도 내 상황에 맞는 결론이 나오지 않음
- 인생의 파트너를 찾는 것보다 어렵다고 느낄 만큼 결정 장벽이 높음

**이들이 모르는 것들:**
- 어떤 기준으로 집을 골라야 하는가
- 집에 대한 나의 가치관은 무엇인가
- 대출은 얼마나 받을 수 있는가
- 관심 지역의 현재 이슈와 미래 전망은 어떤가
- 내 생애 라이프 스테이지를 고려하면 무엇을 더 따져야 하는가

**핵심 pain point:** 정보는 넘치는데 "내 상황에 맞는 결론"이 없다. 집 구매는 단계마다 필요한 정보가 달라서, 어디서부터 시작해야 할지조차 막막하다.

**우리 시스템이 해결하는 것:** 다양한 관점을 가진 페르소나들이 사용자의 질문에 각자의 시각으로 답하고 토론한다. 사용자는 일방적인 정보가 아닌 "복수의 관점"을 보면서 확신을 만들어갈 수 있다.

## Core Values (핵심 가치)

### 1. 출처 있는 숫자만 말한다 (Data with Source)

근거 없이 숫자를 말하는 AI는 위험하다. 출처를 보여주는 AI는 신뢰할 수 있다.

부동산 전문가가 "이 동네 평균 실거래가 7.2억"이라고 말하면 반드시 근거가 따라온다: "국토교통부 실거래가 API, 2025년 1~3월 해당 단지 기준." 출처를 댈 수 없는 정보는 말하지 않는다.

**실천 기준:**
- 모든 수치 응답에 데이터 출처를 명시할 것
- 데이터가 없으면 "이 부분은 데이터가 부족합니다"로 답할 것
- "~일 수 있습니다", "~가능성이 있습니다" 같은 근거 없는 추측 금지

### 2. 각자의 자리에서 발언한다 (Role-Based Expertise)

여러 페르소나가 존재하는 이유는 책임 영역을 분리하기 위해서다. 한 명이 모든 걸 다 말하면 단일 AI와 다를 바 없다.

재무 정보를 주는 페르소나가 인생 조언을 하거나, 경험을 나누는 페르소나가 대출 계산을 하면 페르소나가 무너진 것이다.

**실천 기준:**
- 각 페르소나는 자신의 전문 영역에서만 발언한다
- 페르소나가 자기 영역을 벗어나면 프롬프트를 수정할 것

### 3. 예스맨은 필요 없다 (Constructive Tension)

모든 페르소나가 "사세요"라고 하면, 혼자 검색하는 것보다 위험하다.

**실천 기준:**
- 반드시 1개 이상의 리스크나 반론이 토론에 포함되어야 한다
- 모든 페르소나가 같은 결론을 내면 → 확증편향 방지 챌린지 자동 주입
- 사용자가 "다 좋다고 하네"라고 느끼면 실패

### 4. 대화로 끝나면 수다다 (Decision to Action)

토론이 정리되지 않으면 사용자에게 아무것도 남지 않는다.

**실천 기준:**
- 대화 종료 시 핵심 결정 + 넥스트 액션 자동 생성
- 각 페르소나의 핵심 의견을 1줄로 요약
- "보류"도 명시적 결정이다. 왜 보류인지 이유를 기록

## Design Principles (설계 원칙)

1. **Text-First** — 부동산 데이터는 눈으로 비교해야 한다. 매물 비교표, 대출 시뮬레이션은 시각적으로 봐야 한다
2. **One Scenario, Fully Working** — "마포 아파트 첫 구매 검토" 하나가 처음부터 끝까지 작동하는 5분 데모가 우선
3. **Role Boundary** — 페르소나가 자기 영역을 벗어나면 시스템이 무너진 것
4. **Defensible Differentiation** — ChatGPT는 "~로 알려져 있습니다"라고 말한다. 우리는 국토교통부 API 실거래가에 [출처]를 달아 말한다
