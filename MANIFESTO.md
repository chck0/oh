# 말하는 비서실 — MANIFESTO

## 한 줄 정의
"강남 오피스텔 수익률 3%면 낮은 거 아냐?" 한마디에,
CFO·CSO·투자컨설턴트 세 명의 AI가 국토교통부 실거래 데이터를 놓고 토론하며
투자 의사결정을 돕는 멀티 에이전트 자문 시스템.

## 타겟 사용자
부동산 매수를 앞두고 "이거 사도 되나?" 혼자 고민하는 개인 투자자

## 우리가 검증해야 할 가정 (Demand Evidence)
사업이 망한다면 이 가정들이 틀렸기 때문이다.

1. 개인 투자자는 1명의 결론보다 3명의 토론을 더 신뢰한다. ⚠️ 가장 약한 전제 — 토론 읽기는 인지 부하
2. 공공 실거래가 데이터만으로 의미 있는 토론이 가능하다. ✅ 국토교통부 API 80+ 지역 연동으로 검증 중
3. 같은 LLM으로 진짜 다른 시각의 토론을 만들 수 있다. ✅ CFO/CSO/투자컨설턴트 POV 분리 구조로 실현, 합의 감지 + 확증편향 방지 로직 내장
4. 사용자가 이 서비스에 돈을 낼 의지가 있다. ⚠️ 가장 약한 전제 — 한국 부동산 정보 시장은 무료가 압도

## 현재 구현된 핵심 기능
지역명 입력 → 실거래 데이터 자동 조회 → CFO·CSO·투자컨설턴트 3인 동시 토론 →
스코어카드(100점) + 비서실장 자동 회의록(결정사항 + Next Action)

### 분석 엔진 (정량적 차별점)
- 표면·실질·레버리지 수익률 + IRR + NPV 자동 계산
- Monte Carlo 시뮬레이션 3,000회 (손실 확률, P5~P95 분포)
- 10년 현금흐름 프로젝션 (임대료 성장률·인플레이션·자산 상승률 반영)
- 취득세·보유세·양도소득세 시뮬레이션
- 스트레스 테스트: 최악/기본/최선 3-시나리오

## 핵심 가치
1. **Data with Source** — CFO는 모든 수치에 [출처: ___] 필수. 출처 없는 숫자는 말하지 않는다
2. **Role-Based Expertise** — CFO는 재무만, CSO는 전략·리스크만, 투자컨설턴트는 적합성 자문만
3. **Constructive Tension** — 셋이 같은 결론이면 실패. CSO는 매 응답에 최소 1개 리스크를 제기한다. 합의 감지 시 확증편향 방지 챌린지를 자동 주입한다
4. **Decision to Action** — 토론은 결정·보류사항·Next Action(동사+담당자+기한)으로 끝난다

## 의도적으로 자르는 것
- 회원가입·로그인 (anonymous only)
- 페르소나 커스터마이징
- 청약·전세·월세 모드 (매매 중심)
- 결제·요금제 (무료, 검증 단계)
- 모바일 앱 (웹 only)
- 멀티 LLM 비교

## 설계 원칙
1. **Text-First** — 수익률 3.2% vs 4.1% vs 2.8%는 귀로 듣는 게 아니라 눈으로 비교한다
2. **One Scenario, Fully Working** — "강남 오피스텔" 한 케이스가 처음부터 끝까지 작동하는 5분 데모가 우선
3. **Defensible Differentiation** — ChatGPT는 "~로 알려져 있습니다"라고 말한다. 우리는 국토교통부 API 실거래가에 [출처]를 달아 말한다
