## 🧑‍🤝‍🧑 분석가 (Analyst) — 5인 검증 패널

**정의**: 각각 고유한 페르소나(검증 영역/말투/금지행동)를 가진 LLM 인스턴스. 점찍어둔 매물 한 채를 자기 영역에서 검증·반박한다.

| 키 (코드) | 한국어 이름 | 검증 영역 | 별점 카테고리 |
|---|---|---|---|
| `market_analyst` | 시세 분석가 | 실거래·호가·인근 시세 | 💰 시세 |
| `location_analyst` | 입지 분석가 | 역세권·학군·생활 인프라 | 🏢 입지 |
| `risk_analyst` | 리스크 분석가 | 단지·거시 리스크 | ⚠️ 리스크 |
| `finance_analyst` | 재무 분석가 | 대출 한도·월 상환액·총취득비용 | 💳 재무 |
| `future_analyst` | 미래가치 분석가 | 개발 호재·5~10년 전망 | 🎯 미래가치 |

보조 에이전트
- **MC** (`mc`): 5~6개 짧은 인터뷰 진행
- **서기** (`clerk`): 5인 분석을 별점·합의 결론 한 페이지로 종합

각 에이전트는 `agents/*.md`에 명세된 규칙을 따른다.

## 🔍 Second Opinion (2차 의견) — 본 시스템의 포지셔닝

**정의**: 부동산 중개인 의견에 의존하던 의사결정에 **거래 인센티브에서 자유로운 검증·반박**을 제공해, 사용자가 균형 잡힌 판단을 할 수 있게 하는 역할.

- 우리는 "사세요/사지 마세요"를 말하지 않는다 — 검증·근거를 제공하고, 결정은 사용자에게 돌려준다.
- 부동산 중개인 = 거래 성사가 수수료. 우리 시스템 = 결정의 정확성이 가치.

## 👤 BuyerProfile

**정의**: MC가 5~6개 짧은 질문으로 수집하는 사용자의 검증 입력 프로필.

| 필드 | 설명 | 인터뷰 예시 |
|---|---|---|
| `assets_manwon` | 보유 자산 (대략적 범위) | "한 2억 정도요" |
| `loan_capacity_manwon` | 대출 한도 (총액 O, 월 X) | "3.3억까지요" |
| `office_address` | 회사 위치 (구체적 주소) | "광화문 OO빌딩" |
| `commute_mode` | 출퇴근 수단 | "주로 지하철" |
| `priorities` | 가장 중요한 1~2개 | "자산 가치 + 출퇴근" |

> 본 필드 정의는 Phase 2 인터뷰 단축 작업에서 코드에 반영 예정.

## 📍 검증 대상 매물 (Target Property)

**정의**: 사용자가 점찍어둔 단 1채의 아파트. Scene 03에서 자유 주소 입력 → 행정동/도로명 변환 → 지도 컨펌 후 확정.

- 입력 예: `광명시 OO아파트 105동 1203호`
- 변환·확정 후 5인 분석가의 검증 대상으로 고정

## 🎤 7-Scene 플로우

**정의**: 본 시스템의 핵심 진행 구조 ([`docs/SCENARIO_v1.md`](docs/SCENARIO_v1.md) 참조).

1. **Scene 01 — 진입**: 인트로
2. **Scene 02 — 인터뷰**: 5~6개 짧은 질문
3. **Scene 03 — 매물 주소 입력**: 변환 + 지도 + 컨펌
4. **Scene 04~05 — 분석·토론**: 5인 분석가 다관점 검증
5. **Scene 06 — 종합 리포트**: 별점·합의 결론 한 페이지
6. **Scene 07 — 후속 액션**: 드릴다운 / 채팅 / PDF (3가지 가안)

---

## 📊 출처 명시 (Source Citation)

**정의**: 모든 분석가가 수치를 말할 때 반드시 데이터 근거를 밝히는 규칙.

- 국토부 출처 형식: `[출처: 국토교통부 실거래가 공개시스템, API: getRTMSDataSvcAptTrade, LAWD_CD: {code}, N=?건, 조회일: {date}]`
- 정책대출: `[출처: 한국주택금융공사 2025-Q1]`
- 호재: `[출처: 서울시 도시계획, 발표일: 2024-08, 실현 예정: 2028]` 형식 의무
- 데이터가 없으면 "이 부분은 데이터가 부족합니다"로 답할 것
- **MANIFESTO 핵심 가치 1번** ("출처 있는 숫자만 말한다")의 실천 기준

---

## 📍 P50 실거래가

**정의**: 국토교통부 실거래가 공개시스템 기준 동일 단지·평형 최근 6개월 거래가의 **중앙값(50th percentile)**.

- 호가(매도 희망가)가 아닌 실제 계약 완료된 가격의 중앙값
- 시세 분석가가 "호가 적정성" 평가 시 기준값으로 사용
- 표본 N<5인 경우 "표본 부족 — 신뢰구간 넓음" 경고 의무
- 산출 모듈: `src/molit_api.py` + `src/property_audit.py`

## 🔴 검증 영역 분리 (Verification Boundary)

**정의**: 5인 분석가가 자기 검증 영역에서만 발언하도록 강제하는 규칙. 영역 침범 시 시스템 가치(다관점 검증)가 무너진다.

| 발언 OK | 발언 금지 |
|---|---|
| 시세 → P50, 호가 편차, 분포 | 입지 추천, 대출 계산, 호재 분석 |
| 입지 → 통근, 학군, 인프라 | 가격 평가, 시장 통계, 재무 |
| 리스크 → 단지+거시 리스크 | 호가 평가, 입지 추천 |
| 재무 → LTV/DSR, 월 상환, 정책대출 | 입지·시세·호재 |
| 미래가치 → 호재·악재, 5~10년 전망 | 현재 호가, 월 상환 |

---

## ⭐ 별점 (Star Rating) — Scene 06

**정의**: 5인 분석가가 자기 카테고리에 대해 1~5점 별점으로 검증 결과를 구조화 출력.

- 5점: 매우 긍정적 / 1점: 주의 매우 필요
- 별점 + 한 마디 코멘트 (예: "오를 가능성 ↓", "최상급", "주의 필요")
- 종합 평점 = 5개 별점의 가중·합의 산출 (Phase 4 알고리즘 정의 예정)

**별점 흩어짐 검증**: 5인 별점 표준편차가 너무 낮으면(<0.5) "모두 동의 — 의심 부족" 자동 챌린지 → 재검증 ([`docs/PREMORTEM.md`](docs/PREMORTEM.md) 시나리오 1).

---

## 📝 종합 리포트 (Summary Report) — Scene 06

**정의**: 5인 분석가의 결과를 **한 화면 1페이지에 별점·합의 결론으로 종합한 출력**. 서기(Clerk)가 생성.

필수 요소
- 매물 주소 + 종합 평점 (예: ★★★☆☆ 3.2/5)
- 합의 결론 한 줄 (예: "5명 중 3명이 신중 검토 권합니다")
- 5개 카테고리 별점 + 한 마디 코멘트
- 핵심 쟁점 한 줄 (예: "입지·생활 만족도 vs. 5~10년 미래가치")
- 후속 액션 3가지 안내(드릴다운/대화/PDF)

---

## ✅ 후속 액션 (Follow-up Action) — Scene 07

**정의**: 종합 리포트 이후 사용자가 더 알아볼 수 있도록 돕는 3가지 도구. 결정 자체는 사용자에게 돌려준다.

| 가안 | 설명 |
|---|---|
| **A. 드릴다운** | 별점 카드 클릭 → 재건축 이력·뉴스·유사 사례 |
| **B. 에이전트와 대화** | 채팅창에서 직접 질문, 여러 분석가 다른 관점 응답 |
| **C. PDF 저장·공유** | 한 페이지 요약을 가족·지인과 공유 |

## 🎯 Gold Standard

**정의**: "이 매물·인터뷰가 들어오면 이 5인 응답·종합 리포트가 나와야 한다"는 **기준 출력**. Mock 데모(`demo_mock.py`)가 Gold Standard를 그대로 보여주는 시연용 데이터.

- 데모 시나리오: 광명시 OO아파트 검증 (37세 미혼 직장인 매물)
- 페르소나 명세서(`agents/*.md`)에 카테고리별 예시 응답 포함

---

## 🔴 검증·반박 강제 (Forced Verification)

**정의**: 모든 분석가가 응답에 **반드시 1개 이상의 의심·반박·리스크를 포함**하도록 강제하는 규칙.

- "다 좋아요"만 하면 = 검증 시스템 실패
- 5인 별점이 너무 균일(표준편차 <0.5)하면 자동 챌린지로 재검증
- MANIFESTO 핵심 가치 2번 ("검증·반박이 우선이다")의 실천 기준

---

## 📡 MOLIT API (`src/molit_api.py`)

**정의**: 국토교통부가 제공하는 아파트 실거래 가격 정보 공공 API 연동 모듈.

- API명: `getRTMSDataSvcAptTrade`
- 동일 단지·평형 6개월 P50 산출
- 출처 자동 생성: `[LAWD_CD: {code}, N=?건, 조회일: {date}]`
- 접근: [공공데이터포털](https://www.data.go.kr/) API 키 발급
- API 키 없을 때 샘플 데이터 자동 fallback
- 시세 분석가의 핵심 데이터 소스

## 🗺 Kakao Local API (Phase 3 예정)

**정의**: Scene 03 매물 주소 입력 단계에서 자유 입력 주소를 표준화하고 좌표·지도를 표시하는 외부 API.

- 환경 변수: `KAKAO_REST_API_KEY`
- 용도: 행정동/도로명 → 표준 주소 변환, 지도 핀 표시
- 입지 분석가의 통근 시간·인프라 분석에도 활용 예정

## 🏷 MVP 정의

**한 매물 검증의 7-scene 플로우가 end-to-end로 작동하는 5분 데모.**

1. **Scene 01~02**: 인트로 + 5~6개 인터뷰
2. **Scene 03**: 매물 주소 입력·변환·컨펌
3. **Scene 04~05**: 5인 분석가 다관점 검증
4. **Scene 06**: 종합 리포트 한 페이지 (별점 + 합의 + 핵심 쟁점)
5. **Scene 07**: 후속 액션 안내 (드릴다운/대화/PDF — 최소 1개 가안 작동)

이 흐름이 끊기면 MVP로 인정하지 않는다.
