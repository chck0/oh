# VerifyHome — SLC 설계 문서

기준일: 2026-05-12

---

## 한 줄 요약

> **주소를 입력하면 실거래가 차트와 주변 시세 비교를 보여주고, 챗봇이 그 숫자의 의미를 설명해준다.**

---

## 왜 이게 SLC인가

부동산 앱을 처음 쓰는 사람이 가장 먼저 궁금한 건 딱 하나다.

> "내가 보는 이 아파트, 지금 가격이 적정한가?"

그 질문에 답하는 데 필요한 최소한:

1. **실거래가 데이터** — 최근 몇 달간 실제로 얼마에 팔렸나
2. **비교 기준** — 옆 동네는 얼마인가 (기준이 없으면 숫자는 의미 없다)
3. **해석 도구** — 그래프가 낯선 사람을 위한 말 걸기

이 세 가지만 하고, 완결되게 한다.

---

## 사용자 플로우

```
[page0-web.html]
주소 입력 + 자동완성
  ↓ 아파트 선택 → 평형 선택
  ↓
[market-web.html]
실거래가 차트 (단지 vs 동 평균)
  + 동 비교 추가 (옆 동네 시세 오버레이)
  + 입지 칩 (지하철·학교·병원·마트 등)
  + 혐오시설 경고 (화장장·교도소 등, 있을 때만)
  ↓
챗봇
  "이 아파트 왜 비싸요?" → 입지 데이터 기반 답변
  "옆 동네랑 왜 차이나요?" → 차트 + 입지 비교 답변
  PDF 저장
```

## 기술 스택

| 레이어 | 기술 |
|--------|------|
| 프론트엔드 | HTML/CSS/JS (빌드 없음, 파일 2개) |
| 백엔드 | FastAPI (`src/market_api.py`) |
| LLM | Claude Sonnet (`/api/chat`) |
| 주소 검색 | 도로명주소 API (juso.go.kr) |
| 실거래가 | 국토교통부 실거래가 공공API |
| 입지 데이터 | 정적 JSON 하드코딩 (외부 API 없음) |

## 데이터 인벤토리 (실제 사용)

### 실시간 API 호출

| 데이터 | 출처 | 엔드포인트 |
|--------|------|-----------|
| 주소 자동완성 | juso.go.kr | `/api/address-search` |
| 아파트 매매 실거래가 | 국토교통부 | `/api/real-data` |
| 동 평균 시세 비교 | 국토교통부 | `/api/compare-dong` |
| 주변 편의시설 카운트 | 정적 JSON | `/api/nearby-amenities` |

### 정적 JSON (하드코딩, API 없음)

| 파일 | 내용 | 반경 기준 |
|------|------|----------|
| `subway_stations.json` | 서울 지하철역 263개 | 800m (도보 10분) |
| `elementary_schools.json` | 초등학교 201개 | 1km |
| `secondary_schools.json` | 중고등학교 216개 | 1km |
| `hospitals.json` | 종합·대학병원 40개 | 2km |
| `parks.json` | 공원 114개 | 1km |
| `highway_ics.json` | 고속도로 IC 48개 | 3km |
| `large_stores.json` | 이마트 13 + 백화점·복합몰 22개 | 2~3km |
| `disamenities.json` | 화장장·쓰레기처리장·고압시설·공장·교도소·군부대 41개 | 500m~3km |

## 챗봇 컨텍스트 구조

챗봇이 답할 때 쓰는 데이터:

```
[시세 데이터]
- 단지 최근 평균가, 6개월 변동률
- 동 평균가 대비 단지 프리미엄/할인
- 비교 동 추가 시 해당 동 데이터

[입지 데이터]
- 긍정: 지하철역 N개, 초등학교 N개, 중고등학교 N개,
        병원 N개, 공원 N개, IC N개, 이마트 N개, 백화점 N개
- 부정: 화장장·쓰레기처리장·고압시설·공장·교도소·군부대 (있을 때만)
```
