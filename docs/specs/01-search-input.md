# Spec: 검색 조건 입력 화면

> **상태**: Implemented  
> **구현 파일**: `web/search.html`  
> **작성일**: 2026-05-24 (retrospective)

---

## 1. Why

MANIFESTO — "직장 주소 하나를 넣으면, 친구처럼 솔직하게 말해준다."  
검색의 시작점은 집이 아닌 직장이어야 한다 (Why Tree #1).  
사용자가 거주 조건을 5단계로 좁혀, 자신에게 맞는 매물만 받을 수 있게 한다.

---

## 2. User Story

```
As a 직장인,
I want to 직장 주소와 통근시간·예산·평형 조건을 입력하고,
so that 내 조건에 맞는 아파트 추천 결과를 바로 받는다.
```

---

## 3. Scope

### In-scope
- 직장 주소 입력 (Daum Postcode 팝업)
- 최대 매매가 입력 (억 + 만원 분리 입력)
- 최대 통근시간 선택 (10·20·30·40·50·60분 칩)
- 관심 평형 선택 (복수, 체크박스 칩)
- 단지 세대수 최소치 선택 (라디오, 선택사항)
- 폼 제출 → `result.html?쿼리파라미터` 이동

### Out-of-scope
- 로그인 / 회원 정보
- 검색 히스토리 저장
- 전세·월세 조건
- 지역 직접 선택 (직장 주소 기반이 원칙)

---

## 4. Functional Requirements

- F1. 직장 주소는 Daum Postcode SDK로 도로명주소 또는 지번주소 선택
- F2. 최대 매매가: 억(정수) + 만원(0·1000·2000…9000 천만 단위) 분리 입력, 합산 ≥ 10,000만원(1억)
- F3. 통근시간: 10~60분 칩 단일선택, 기본값 60분
- F4. 평형: `10평미만·10평대·20평대·30평대·40평대·50평대+` 복수선택, 기본값 20평대+30평대
- F5. 세대수 최소치: 제한없음·300·500·1,000·1,500세대 라디오, 기본값 제한없음(내부 기본 100)
- F6. 폼 제출 시 URL 파라미터로 `result.html` 이동
  - `workplace_address`, `max_price`(만원), `max_minutes`, `pyeong_types`(쉼표), `min_kaptdaCnt`(선택)
- F7. 필수 항목 미입력 시 인라인 에러 메시지 (alert 사용 금지)

---

## 5. Non-functional Requirements

- **성능**: 페이지 로드 < 1초 (정적 HTML)
- **모바일**: max-width 720px 중앙 정렬, 터치 칩 선택 편의
- **보안**: 주소는 Daum SDK 팝업 → readonly input (직접 입력 차단)

---

## 6. UX / Vibe

MANIFESTO — "분석 리포트가 아니다. 카톡 한 줄이다."  
입력 단계도 군더더기 없이 5단계만. 각 섹션에 ⓘ 팝오버로 보충 설명.  
제출 버튼: "분석하기 →" (약 30초~1분 소요 안내 포함)

---

## 7. Data Model

없음 (프론트 전용). URL 파라미터로 result.html에 전달.

| 파라미터 | 타입 | 예시 |
|---|---|---|
| `workplace_address` | string | "서울 강남구 테헤란로 504" |
| `max_price` | int (만원) | 50000 |
| `max_minutes` | int | 60 |
| `pyeong_types` | string (쉼표) | "20평대,30평대" |
| `min_kaptdaCnt` | int (선택) | 300 |

---

## 8. API / Interface

없음. 폼 제출 시 `window.location.href = '/result.html?' + params` 이동.

---

## 9. Edge Cases

| 케이스 | 현재 동작 |
|---|---|
| 직장 주소 미입력 | 인라인 에러 메시지 표시, 제출 차단 |
| 금액 1억 미만 | 인라인 에러 표시, 제출 차단 |
| 만원 칸 천만 단위 아님 | 인라인 에러 표시 |
| 평형 0개 선택 | alert("평형을 하나 이상 선택해주세요") |
| 모바일 소형 화면 | 칩 wrap, 최대폭 100vw |

## 10. Acceptance Criteria

- [x] 직장 주소 Daum Postcode 팝업으로 입력
- [x] 금액 1억 미만 입력 시 에러
- [x] 통근시간 칩 단일선택 (기본 60분)
- [x] 평형 복수선택 (기본 20·30평대)
- [x] 세대수 라디오 선택
- [x] 제출 시 URL 파라미터 포함 result.html로 이동
- [x] 필수 항목 미입력 시 진행 차단
