# Spec 41: POI 마커 매칭 완화 (정상 시설 오거부 수정)

> 상태: Implemented | 작성일: 2026-06-15 | 브랜치: hjkang83

## 1. Why (왜)

- **버그:** spec-39(이름 검증) 이후, 지도에 뻔히 있는 **"안암역 3번출구"·"고려대"** 같은 시설을 클릭해도 "지도에서 위치를 확인하지 못했습니다" 안내만 뜨고 핀이 안 찍힘. 이런 오거부가 자주 발생.
- **원인 2가지:**
  1. **거리 하드필터** — `_pickBestPoi`가 후보의 직선거리를 `distM*1.5+200m`로 제한. 그런데 **넓은 시설(대학 캠퍼스·지하철 역사)** 은 Kakao 등록 좌표가 *중심점*이라, 도보 기준점(정문·출구)과 수백 m 이상 떨어질 수 있음 → 정상 시설이 거리에서 탈락.
  2. **검색 반경이 좁음** — `searchRadius = distM+300`(최대 2000m). 큰 시설의 등록 좌표가 반경 밖이면 keywordSearch가 0건 → 폴백도 좁아 실패.
  - 추가로 폴백 질의 `substring(0,6)`이 다단어명("안암역 3번출구")에 부적합.
- **유지해야 할 것:** spec-39가 고친 오핀(홍제초→인왕초)은 막아야 함. 이는 **이름 검증(정확/부분 일치)** 만으로 충분히 차단됨(서로 이름에 포함 안 됨).

## 2. Scope (범위)

- **포함:**
  - `_pickBestPoi`에서 **거리 하드필터 제거**. 이름 정확/부분 일치 + 거리순(Kakao 정렬) 최근접만으로 선택.
  - 검색 반경 확대(넓은 시설 포함).
  - 폴백 질의 개선: 다단어명은 **첫 단어**로 재검색(예 "안암역").
- **제외(안 함):**
  - 이름 무검증 최근접(data[0]) 배치 — 여전히 금지(오핀 방지).
  - POI 좌표 DB 저장(spec-39 후속) · 백엔드/API 변경.

## 3. 설계 (어떻게)

- **건드리는 파일:** `web/result.html` 뿐.
- **DB/API 변경:** 없음

- **`_pickBestPoi(data, poiName)`** (시그니처 단순화: distM/좌표 인자 제거):
  - data는 location 기준 거리순. 앞에서부터 **정확 일치** 우선, 없으면 **부분 일치**(한쪽이 다른쪽 포함) 최근접. 없으면 `null`.
  - **거리 검증 안 함** — 지리 제약은 검색 반경이 담당.
  - `_haversine` 제거(미사용).
- **검색 반경:** `min(max(distM*2 + 1500, 2000), 8000)` — 넓은 시설 포함, 타 지역 오매칭은 차단.
- **폴백 질의:** 첫 공백 토큰이 있고 2자 이상이면 그것으로, 아니면 `substring(0,6)`. 풀네임 실패 시 1회 재검색.

- **엣지케이스:**
  | 케이스 | 동작 |
  |---|---|
  | 안암역 3번출구 | 풀네임 매칭, 실패 시 "안암역" 폴백 → 부분 일치 |
  | 고려대(학교) | 부분 일치(한쪽 포함), 반경 확대로 캠퍼스 좌표 포함 |
  | 홍제초 vs 인왕초 | 이름 정확 일치로 홍제초 선택(오핀 방지 유지) |
  | 체인점(CU 등) | 이름 일치 최근접 = 가장 가까운 지점 |
  | 진짜 못 찾음 | 핀 미배치 + 안내 + 활성표시 해제 |
