# Spec 36: 대출 가능액 역산 계산기 (Affordability Calculator)

> 상태: Approved | 작성일: 2026-06-13 | 브랜치: claude/loan-affordability

## 1. Why (왜)

- **문제:** 현재 검색의 "가격 범위 → 최대"는 사용자가 **매수가능액을 숫자로 직접** 입력해야 한다. 그런데 첫 집 구매자가 실제로 아는 건 "내 현금 얼마 + 소득 얼마"지, "그래서 몇 억짜리까지 살 수 있나"가 아니다. 이 변환을 머릿속으로/외부 계산기로 하느라 검색 진입이 막힌다.
- **사용자가 얻는 것:** 보유 현금·연소득(+금리)을 넣으면 **"대략 N억까지"**를 즉석에서 역산해, 최대 가격 칸을 자동으로 채워준다. "얼마짜리를 봐야 하지?"의 1차 답.
- **데이터/비용:** 순수 산수. 새 API·새 DB·LLM 호출 0.

### 매니페스토 정합성 (중요)
- AGENTS.md 타겟은 "예산 범위를 **대략은 아는** 30대 1인 가구"이고, "예산 모르는 완전 초보 단계"는 의도적 제외다. → 본 기능은 **예산을 정밀 산정하는 사전심사 플로우가 아니라**, 대략 아는 사람이 숫자 칸을 빠르게 채우도록 돕는 **선택적 보조 위젯**으로 한정한다.
- "데이터 없으면 단정하지 않는다" → LTV/DSR/금리는 정책·개인 신용에 따라 달라지므로, 결과에 **"추정치 · 실제 한도는 은행 확인 필요"**를 항상 병기한다. 단정 표현 금지.

## 2. Scope (범위)

확정(2026-06-13 사용자 결정): ① **현행 법규 기반**(규제지역 지정·LTV·DSR·현행 금리), ② **별도 섹션**, ③ 결과는 **최대 가격만** 채움(최소는 제한없음 유지 — 추천).

- **포함:**
  - search.html에 **신규 독립 섹션** "💡 얼마까지 살 수 있나" (가격 범위 섹션 위, 기본 접힘)
  - 입력: 보유 현금(억/만원), 연소득(만원/년), **지역(비규제/규제 토글)**, **생애최초 여부**, (선택) 실금리(%)·대출기간(년)
  - 계산: 현행 LTV·절대 대출한도 캡·스트레스 DSR 한도 중 **가장 보수적인(작은) 값**을 매수가능액으로 역산
  - "이 금액을 최대 가격으로" 버튼 → 기존 `price-eok`/`price-man`에 주입
  - **"2026-06 기준 추정 · 규제·금리 수시 변동 · 은행 확인 필요"** 출처 링크와 함께 상시 노출
- **제외(안 함):**
  - 백엔드·검색 API 변경 (순수 프론트 위젯, 기존 `max_price` 흐름 재사용)
  - 실시간 금리 API·우대금리·신용등급 연동, 보금자리·디딤돌 등 정책상품 자동판별
  - 다주택자 LTV 차등(0%~)·DTI·소득증빙 종류·기존대출 차감(=DSR 잔여한도) 정밀 반영
  - 자동 검색 실행 (값만 채우고 검색은 사용자가 누름)

## 3. 설계 (어떻게)

- **건드리는 파일:** `web/search.html` 만 (신규 섹션 HTML + JS 계산 + CSS). 백엔드 무관.
- **DB 변경:** 없음 · **API 변경:** 없음 (기존 `max_price` 그대로)

### 현행 규제 상수 (2026-06 기준, JS 상단 `AFFORD` 객체로 한 곳에 모음 — 수시 변동이라 갱신 쉽게)
출처: 금융위 6·27 대책 보도자료, 은행 비교 자료(아래 §6).
- **LTV**
  - 비규제지역 무주택: **70%**
  - 규제지역(투기과열·조정지역: 강남3구·용산 등): LTV + **절대 대출한도 캡** 동시 적용
    - 주택가 15억 이하 → 대출 **6억** 상한 / 15억~25억 → **4억** / 25억 초과 → **2억**
    - 생애최초(수도권·규제): LTV **70%**(’25 강화, 종전 80%) + 전입의무
- **DSR:** 40%(1금융권). **스트레스 DSR 3단계** 전면 시행 → 심사금리 = 실금리 + **스트레스 가산 1.5%p**(보수적 고정)
- **기본 실금리:** **4.5%**(사용자 조정 가능), **대출기간:** 30년

### 계산 로직 (역산, 단위 만원)
```
입력: cash, income, region(비규제|규제), firstHome(bool), rate(기본 4.5), years(기본 30)
loanRate = rate + 1.5(스트레스)         # DSR 심사용

① LTV 한도 → 살 수 있는 집값
   비규제: maxByLtv = cash / (1 - 0.70)
   규제:   LTV(생애최초 0.70 / 그 외 0.50 가정)로 maxByLtvRaw = cash/(1-ltv)
           단, 대출액 = price-cash ≤ 캡(6/4/2억, price 구간별) → 캡으로 재클램프
           maxByLtv = min(maxByLtvRaw, cash + 캡)

② DSR 한도 → 갚을 수 있는 대출 → 집값
   월상환여력 = income × 0.40 / 12
   대출가능액 = 원리금균등역산(월상환여력, loanRate/12, years×12)
   maxByDsr = cash + 대출가능액

③ 매수가능액 = min(maxByLtv, maxByDsr)  (둘 중 작은 값 = 실제 제약)
   → 만원 단위 내림, 검색 상한(200억) 클램프, 억/만원 분해
```
- 원리금균등 역산: `대출 = 월납 × (1-(1+r)^-n)/r`, r=0이면 `월납 × n`(0 division 방지).
- 규제지역 LTV(50%)와 절대캡은 가정값 — 정확값은 주택가/주택수/생애최초에 따라 달라지므로 패널에 "가정" 명시.

- **엣지케이스:**
  | 케이스 | 동작 |
  |---|---|
  | 현금·소득 둘 다 공란/0 | 계산 버튼 비활성 + "현금·소득을 입력하세요" |
  | 소득만 0 | LTV 한도만으로 추정(+"소득 미입력 → DSR 미반영" 안내) |
  | 현금만 0 | DSR 한도만(+"현금 미입력 → 대출만 가정") |
  | rate=0 | 무이자 단순분할, 0 division 방지 |
  | 결과 > 200억 | 200억 클램프 |
  | 모바일 | price-row 레이아웃 재사용, 접이식 |

- **UX/톤:** 카톡 친구 톤. 예: "현금 1.5억 + 연소득 5천이면 비규제 기준 **대략 4.8억까지** 볼 만해. 추정치니까 실제 한도는 은행에서 꼭 확인해." 이모지 ≤1, 단정 금지.

## 4. 완료 조건 (Acceptance Criteria)

- [x] AC1: 가격 범위 위에 **독립 섹션** 계산기가 있고 기본 접힘이다 (단계 번호 1~5 유지)
- [x] AC2: 현금·연소득·지역·생애최초(+금리) 입력 후 매수가능액이 억/만원으로 표시된다 (LTV·캡·DSR 중 최소). 입력 변경 시 자동 재계산
- [x] AC3: 규제지역 선택 시 LTV 50%·절대 대출한도 캡이 반영되어 비규제보다 보수적으로 나옴 (라이브: 비규제 4.2억 → 규제 3억 확인)
- [x] AC4: "최대 가격으로" 누르면 `price-eok`/`price-man`이 그 값으로 채워짐 (라이브 확인)
- [x] AC5: "2026-06 기준 추정 · 수시 변동 · 은행 확인 필요" + 금융위 출처 링크 상시 노출 (단정 없음)
- [x] AC6: 공란/0/이상치에 깨지지 않고 결과 숨김 + "현금·소득 미입력" 가정 안내
- [x] AC7: 순수 프론트(search.html) — 백엔드/검색 API 변경 없음, 기존 검색 흐름 정상
