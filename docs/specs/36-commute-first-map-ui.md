# 36. 통근시간 중심 지도형 결과 UI

## Why

검색 결과 화면에 가격, 태그, AI 코멘트, 통계, 버킷 정보가 한 번에 노출되어 첫 화면 가독성이 떨어진다.
BADUGI의 포지셔닝은 "직장 기준으로 살 만한 단지 찾기"이므로, 지도와 목록의 첫 숫자는 가격보다 통근시간이어야 한다.

호갱노노처럼 지도 위 숫자 라벨을 중심으로 빠르게 훑고, 상세 정보는 선택 이후에 확인하는 흐름을 따른다.

## Scope

- `web/result.html`의 결과 목록 카드 정보량을 줄인다.
- 지도 핀의 주 표시값을 가격에서 통근시간으로 바꾼다.
- 가격, Why-price 태그, AI 친구 코멘트는 삭제하지 않고 상세/보조 정보로 낮춘다.
- 백엔드 API 응답, DB 스키마, 검색 파이프라인은 변경하지 않는다.

## 설계

### 지도 핀

- 단지별 대표 핀은 가장 빠른 통근시간 카드를 기준으로 정렬한다.
- 핀 첫 줄은 `NN분`으로 표시한다.
- 지도 핀은 가시성을 위해 `NN분` 숫자만 크게 표시한다.
- 평형/가격은 지도 핀에서 빼고 호버 툴팁과 좌측 목록에서 확인한다.
- 30분 이하, 45분 이하, 그 외 구간은 색으로 구분한다.
- 색상 의미는 지도 위 오버레이 범례로 명시한다.
- 추천 단지는 기존 `pin-best` 강조를 유지한다.
- 여러 평형이 있으면 기존 평형 수 마커를 유지한다.

### 좌측 목록

- 추천 카드는 통근시간을 가장 큰 숫자로 표시한다.
- 가격은 보조 메타 행으로 낮춘다.
- Why-price 태그는 1~2개만 노출한다.
- AI 친구 코멘트는 목록에서 숨기고, 상세 패널/툴팁에서 확인한다.
- 일반 리스트 카드도 동일하게 통근시간 우선 구조로 바꾼다.

### 상단 툴바

- 정렬순서는 조건 요약 아래 별도 줄이 아니라 조건 요약과 같은 상단 라인에 배치한다.
- 화면 폭이 좁으면 정렬 컨트롤은 같은 툴바 안에서 다음 줄로 줄바꿈된다.

### 상세 유지

- 카드 클릭, 핀 클릭, 경로 상세, 즐겨찾기, 비교 기능은 기존 동작을 유지한다.
- 상세 패널의 AI 코멘트와 가격 분석은 그대로 유지한다.

## 완료 조건

- 검색 결과 지도 핀에서 가격보다 통근시간이 먼저 보인다.
- 지도 위에서 핀 색상 기준을 바로 확인할 수 있다.
- 정렬순서 컨트롤이 조건 요약과 같은 상단 툴바에 표시된다.
- 좌측 추천/일반 카드의 주 숫자가 통근시간이다.
- 좌측 목록에서 카드당 정보량이 줄어 스크롤 밀도가 개선된다.
- `.result-layout` 자식 구조를 추가/변경하지 않는다.
- 기존 즐겨찾기, 경로 보기, 상세 열기 이벤트가 유지된다.
