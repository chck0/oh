---

## 1. Why (왜 만드는가)

- 친구 한마디는 한 줄 코멘트에 그쳐 사용자가 더 깊은 질문을 할 수 없었음
- "호가가 얼마야?", "전세가는?" 같은 후속 질문이 자연스럽게 이어질 수 없는 UX
- "부동산 잘 아는 친구"라는 서비스 컨셉을 대화형으로 확장해 차별화

---

## 2. User Story

```
As a 매물을 검토 중인 사용자,
I want to  친구 한마디 카드를 탭해서 해당 아파트에 대해 자유롭게 질문하고,
so that    호가·전세가·학군·투자가치 등 궁금한 내용을 실시간으로 물어볼 수 있다.
```

---

## 3. Scope

### In-scope
- 친구 한마디 카드(추천 카드 / 좌측 패널 detail) 클릭 시 채팅 패널 오픈
- Claude Opus 4-8 모델 사용
- 예시 질문 칩 5개 제공
- 대화 히스토리 유지 (최대 10턴)
- 단지 실거래 데이터를 컨텍스트로 제공

### Out-of-scope
- 실시간 호가/전세 외부 API 연동 (호갱노노, 네이버부동산 등)
- 대화 내용 DB 저장/히스토리 복원
- 스트리밍 응답
- 모바일 전용 UI 최적화

---

## 4. Functional Requirements

### F1. 채팅 패널 오픈
- 친구 한마디 카드 클릭 → 지도 오른쪽에 슬라이드 패널 등장
- 패널에 단지명 표시, 예시 질문 칩 5개 노출
- 첫 질문 전까지 칩 표시, 이후 숨김

### F2. 예시 질문 칩
1. 지금 호가가 얼마야?
2. 전세가는 어떻게 돼?
3. 학군은 어때?
4. 투자 가치 있어?
5. 단지 장단점 솔직하게 말해봐

### F3. Claude 호출 (`POST /api/apt/{apt_seq}/chat`)
- **모델**: `claude-opus-4-8`
- **컨텍스트**: 단지명, 위치, 세대수, 준공연도, 최근 3년 실거래 20건
- **페르소나**: 부동산 잘 아는 친구, 반말, 카톡 말투, 5줄 이내
- **실시간 데이터 없음 명시**: 호가·전세 추정 시 "확인 필요" 필수
- **히스토리**: 최대 10턴(20 messages) 전달

### F4. 채팅 UI
- 오른쪽 고정 패널 (width: 370px, height: 100vh - 54px)
- 사용자 메시지: 오른쪽 정렬, 브랜드 컬러
- AI 응답: 왼쪽 정렬, 회색 배경
- Enter 키 / 전송 버튼으로 발송
- 응답 대기 중 스피너 표시

## 5. Data Model

### 신규 API
```
POST /api/apt/{apt_seq}/chat
Body: { pyeong_type, wp_id, message, history: [{role, content}] }
Response: { reply: string }
```

### 컨텍스트로 사용하는 기존 테이블
- `apartments`: apt_nm, umd_nm, kaptdaCnt, build_year
- `trade_history`: 최근 3년 실거래 20건

### 주요 결정 사항

1. **Claude Opus 사용**: 채팅은 깊은 맥락 이해가 필요해 Opus 선택 (일반 카드 Haiku, 추천 카드 Sonnet과 구분)
2. **실시간 데이터 없음 솔직히 명시**: 호갱노노·네이버 API 미연동 → 프롬프트에서 추정 시 "확인 필요" 강제
3. **동기 Anthropic 클라이언트**: FastAPI sync 엔드포인트에서 호출 (async 엔드포인트 변환 불필요)
4. **히스토리 10턴 제한**: 컨텍스트 과부하 방지

### 알려진 제약

- 실시간 호가/전세 정보 없음 → Claude 추정값 + "확인 필요" 안내
- Vercel 60초 타임아웃: Claude Opus 응답이 10~15초 소요될 수 있음
- 대화 내용 DB 미저장 → 새로고침 시 히스토리 초기화
