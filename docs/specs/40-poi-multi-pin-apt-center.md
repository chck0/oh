# Spec 40: 도보 시설 다중 핀 유지 + 아파트 정중앙 고정

> 상태: Implemented | 작성일: 2026-06-15 | 브랜치: hjkang83

## 1. Why (왜)

- **문제:** 상세 패널의 "도보 주요시설"을 클릭하면 지도에 핀이 1개만 찍히고, 다른 시설을 누르면 **이전 핀이 사라진다**. 또 클릭한 시설로 지도가 이동해 **아파트가 화면에서 벗어난다**. 그래서 여러 시설의 거리를 한눈에 비교할 수 없다.
- **사용자가 얻는 것:** 아파트를 지도 정중앙에 고정한 채, 클릭한 시설들의 핀이 **누적되어 남는다**. 동일 시설을 다시 누르면 그 핀만 사라진다(토글). → 클릭한 시설들의 거리·방향을 한눈에 파악.

## 2. Scope (범위)

- **포함:**
  - 도보 시설 클릭 시 핀을 **누적**(다중) 표시. 이전 핀 유지.
  - 같은 시설 재클릭 → 해당 핀만 제거(토글). 행의 활성 표시도 동기.
  - 매 클릭 시 **아파트를 지도 정중앙에 고정**, 활성 핀이 모두 보이도록 줌 자동 조정(아파트 기준 대칭 범위).
  - 단지 전환/패널 닫기 시 모든 POI 핀 초기화.
- **제외(안 함):**
  - POI 좌표 DB 저장(spec-39 후속) · 백엔드/API 변경.
  - 핀 디자인 변경(기존 `poi-custom-pin` 재사용).

## 3. 설계 (어떻게)

- **건드리는 파일:** `web/result.html` 뿐.
- **DB/API 변경:** 없음

- **상태:** 단일 `_poiMarker`/`_activePoiRow` → **`_poiMarkers: Map(row→{overlay,pos})`** (다중).
- **`togglePoiMarker(row)`:** 이미 있으면 해당 overlay 제거 + Map에서 삭제 + `poi-active` 해제 + 재정렬. 없으면 `poi-active` 부여 후 `showPoiMarker`.
- **`showPoiMarker(row, ...)`:** spec-39 이름 검증 그대로. 배치 시 `_poiMarkers.set(row, {overlay,pos})` 후 `_recenterApt()`. 검색 실패 시 `poi-active` 해제(핀 없이 활성만 남지 않게).
- **`_recenterApt()`:** 아파트를 중심으로 활성 POI를 모두 포함하는 **상하·좌우 대칭 범위**를 `setBounds` → 아파트가 항상 정중앙. 활성 0개면 아파트로 `setCenter`만.
- **`clearPoiMarker()`:** 전체 제거(단지 전환/닫기).

- **엣지케이스:**
  | 케이스 | 동작 |
  |---|---|
  | 여러 시설 연속 클릭 | 핀 누적, 아파트 중앙 유지, 전체 보이게 줌 |
  | 같은 시설 재클릭 | 해당 핀만 제거 + 남은 핀에 맞춰 재정렬 |
  | 마지막 핀 제거 | 아파트 중심으로 `setCenter` |
  | 위치 확인 실패(spec-39) | 핀 미배치 + 안내 + 활성표시 해제 |
  | 단지 전환/패널 닫기 | 모든 핀 초기화 |
