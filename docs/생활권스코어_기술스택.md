# 생활권 스코어 서비스 — 기술 스택

---

## 전체 아키텍처 개요

```
[사용자 브라우저]
      │
      ▼
[Frontend] Next.js
      │
      ▼
[Backend API] FastAPI (Python)
      │
   ┌──┴──────────────┐
   ▼                 ▼
[DB]           [Data Pipeline]
PostgreSQL      Python 수집기
+ PostGIS            │
                     ▼
              [외부 API / 공공데이터]
              카카오, 에어코리아 등
```

---

## Frontend

| 항목 | 기술 | 선택 이유 |
|---|---|---|
| 프레임워크 | **Next.js 14** | SSR로 지도/점수 초기 로딩 빠름 |
| 스타일 | **Tailwind CSS** | 빠른 UI 구성 |
| 지도 | **카카오맵 SDK** | 한국 주소/좌표 정확도 최고 |
| 차트 | **Recharts** | 레이더 차트 구현 용이 |
| 상태관리 | **Zustand** | 가중치 슬라이더 상태 관리 |

```
핵심 화면 3개
├─ 지도 화면    → 카카오맵 위에 히트맵 오버레이
├─ 단지 상세    → 레이더 차트 + 카테고리별 점수
└─ 비교 화면    → 단지 A vs B 나란히 비교
```

## Backend

| 항목 | 기술 | 선택 이유 |
|---|---|---|
| 프레임워크 | **FastAPI (Python)** | 데이터 처리 라이브러리와 같은 생태계 |
| 점수 계산 | **NumPy / Pandas** | 거리 기반 감쇠 함수 계산 |
| 지리 연산 | **GeoPandas / Shapely** | 반경 내 POI 개수 계산 |
| 인증 | **JWT** | 사용자 프리셋 저장용 |
| API 문서 | **Swagger (자동생성)** | FastAPI 기본 제공 |

```python
# 점수 계산 핵심 로직 예시
def calc_distance_score(distance_m: float) -> float:
    if distance_m <= 100:   return 100
    elif distance_m <= 300: return 80
    elif distance_m <= 500: return 60
    else:                   return 0
```

## Database

| 항목 | 기술 | 선택 이유 |
|---|---|---|
| 메인 DB | **PostgreSQL + PostGIS** | 위경도 기반 반경 쿼리 필수 |
| 캐시 | **Redis** | 자주 조회되는 단지 점수 캐싱 |
| ORM | **SQLAlchemy** | Python 생태계 표준 |

```sql
-- PostGIS 반경 쿼리 예시: 특정 아파트 500m 내 편의점 개수
SELECT COUNT(*)
FROM poi_facilities
WHERE category = '편의점'
  AND ST_DWithin(
    location::geography,
    ST_MakePoint(127.0276, 37.4979)::geography,
    500  -- 반경 500m
  );
```

## Data Pipeline (데이터 수집)

| 항목 | 기술 | 선택 이유 |
|---|---|---|
| 수집 언어 | **Python** | 공공 API 연동 라이브러리 풍부 |
| 스케줄링 | **APScheduler** | 주 1회 자동 갱신, 가볍고 설정 쉬움 |
| HTTP 요청 | **httpx / requests** | API 호출 |
| 데이터 처리 | **Pandas** | CSV 파싱, 정제 |

```
수집 스케줄
├─ 매일   → 미세먼지 (에어코리아 API)
├─ 주 1회 → 편의시설 POI (카카오 로컬 API)
├─ 월 1회 → 범죄통계, 학교정보 (공공데이터포털)
└─ 분기 1회 → 소음지도, 도시계획 (국토부)
```

## 연동 외부 API

| API | 제공 데이터 | 과금 여부 |
|---|---|---|
| **카카오 로컬 API** | 편의시설 POI (마트, 카페, 병원 등) | 무료 (월 30만건) |
| **공공데이터포털** | 범죄통계, 학교정보, 대중교통 | 무료 |
| **에어코리아 API** | 미세먼지, 대기질 측정 | 무료 |
| **국토교통부 API** | 실거래가, 건물정보 | 무료 |
| **카카오맵 SDK** | 지도 렌더링, 좌표 변환 | 무료 (월 30만건) |

## 인프라 / 배포

| 항목 | 기술 | 선택 이유 |
|---|---|---|
| 컨테이너 | **Docker + Docker Compose** | 로컬 개발 환경 통일 |
| Frontend 배포 | **Vercel** | Next.js 최적화, 무료 티어 |

| DB 호스팅 | **AWS RDS (PostgreSQL)** | PostGIS 확장 지원 |
| 캐시 호스팅 | **AWS ElastiCache (Redis)** | RDS와 같은 VPC |

## 기술 스택 요약 (한 눈에)

```
Frontend   : Next.js + Tailwind + 카카오맵 SDK + Recharts
Backend    : FastAPI (Python) + GeoPandas
Database   : PostgreSQL + PostGIS + Redis
Pipeline   : Python + APScheduler + httpx
배포       : Vercel (FE) + AWS EC2/RDS (BE)
외부 API   : 카카오 로컬 + 공공데이터포털 + 에어코리아
```

---

## 개발 순서 (Phase별)

```
Phase 1 — 데이터 파이프라인 (2~3주)
   └─ 카카오 로컬 API 연동 → POI 수집 → PostgreSQL 저장

Phase 2 — 점수 계산 엔진 (2주)
   └─ 거리 감쇠 함수 → 카테고리 점수 → 최종 합산

Phase 3 — Backend API (2주)
   └─ 단지 좌표 입력 → 점수 반환 API 개발

Phase 4 — Frontend (3~4주)
   └─ 지도 화면 → 단지 상세 → 비교 화면

Phase 5 — 사용자 맞춤화 (1~2주)
   └─ 가중치 슬라이더 → 실시간 점수 재계산
```
