# LLM Policy

부동산 검증 AI 에이전트 — LLM 사용 *허용/금지 영역* + 안전 패턴 + 검증 절차

기준일: 2026-05-06

---

## 한 줄 요약

> **LLM은 *고도로 제약된 부분*에서만 사용**한다. *결정*(점수·가중치·트리거)은 룰. *자연어 표현 다듬기·분류·단답*만 LLM 허용. RAG 사용해도 hallucination은 0이 아니므로 위험 영역에서는 절대 금지.

---

## 1. 정책 원칙

### 원칙 1 — 룰 우선, LLM 부가

시스템의 *모든 결정* (점수 산출·가중치 산출·트리거 판정·인과 추론)은 **룰 기반**. LLM은 결과의 *자연어 표현·분류*에만 사용.

### 원칙 2 — 검증 가능한 출력만

LLM 출력은 *반드시 검증 가능*해야 함:
- 분류: 사전 정의된 옵션 중 선택 (regex 매칭 검증)
- 플레이스홀더: 단어 길이·종류 제약 (validation 함수)
- 자유 형식 발언 → ❌ 금지

### 원칙 3 — 매번 같은 결과 노력

- `temperature = 0`
- 동일 입력 → 거의 동일 출력 (100% 보장은 안 됨, LLM 한계)
- 시연 전 *동일 케이스 10회* 실행해 일관성 검증

### 원칙 4 — Hallucination 가드

RAG로 컨텍스트 제공해도 6가지 hallucination 발생 가능:
- 외삽 (없는 정보 추론)
- 단위·맥락 오해
- 숫자 변형
- 표현 과장/축소
- 충돌 정보 임의 선택
- 자기 학습 데이터 섞기

→ *위험 영역*에서는 LLM 금지. *허용 영역*에서도 검증 필수.

## 2. LLM 사용 ✅ 허용 영역

### 2.1. 분류 (사전 정의 옵션 중 선택)

**예시**:
```
입력: "회귀 잔차 +18%, 이 매물 평당가 4,200만원 vs 회귀 적정가 3,560만원"
프롬프트: "이 잔차를 다음 중 분류: [거품 강함, 거품 보통, 적정, 저평가 약함, 저평가 강함]"
출력: "거품 강함"
```

**검증**: 출력이 옵션 5개 중 하나인지 자동 확인.

### 2.2. 자연어 표현 다듬기 (사실 변경 없이)

**예시**:
```
입력 (룰로 생성된 텍스트):
  "광명초등학교가 도보 5분 거리. 학교 인근 매물."

프롬프트: "다음 문장을 더 자연스럽게 다듬되, 모든 숫자·고유명사를 유지:
  '광명초등학교가 도보 5분 거리. 학교 인근 매물.'"

출력 (LLM):
  "도보 5분 거리에 광명초등학교가 있어 학군 매물로 분류됩니다."
```

**검증**: 핵심 키워드(학교명·거리·5분) 모두 포함하는지 regex 확인.

### 2.3. 결정 가이드 메시지 (한 줄, 매우 제약된)

**예시**:
```
입력: "종합 점수 ★3.85, 사용자 우선순위 (미래가치 30%)"
프롬프트: "다음 형식으로 한 줄 가이드 작성:
  '당신의 우선순위(미래가치 30%)에선 [평가] 매물입니다. 매수 결정은 본인이.'
  [평가] 옵션: 검토 가능, 신중 검토, 재검토 필요"
출력: "당신의 우선순위(미래가치 30%)에선 검토 가능 매물입니다. 매수 결정은 본인이."
```

**검증**: 출력 길이 < 100자, [평가] 옵션 3개 중 선택, 사용자 가중치·점수 일치.

### 2.4. 토론 발언 *플레이스홀더만* 채우기

**예시**:
```
사전 템플릿:
  "🟢 옹호: {매물 강점 한 문장}. {수치 인용}이 그 근거."
  "🔴 반대: {매물 약점 한 문장}. {수치 인용}이 그 근거."

룰 산출 결과:
  강점 = "학교 도보 5분"
  강점 수치 = "도보 5분"
  약점 = "노후도 25년"
  약점 수치 = "준공 2001년"

LLM 작업:
  → {매물 강점 한 문장} = "학교 도보 5분으로 학군 매물 가산점 적용"
  → {수치 인용} = "도보 5분"
  
검증: regex로 핵심 단어 일치 확인
```

**검증**: 템플릿 구조 유지, 핵심 데이터 변형 없음.

## 3. LLM 사용 ❌ 금지 영역

### 3.1. 점수 계산

| 작업 | 사유 |
|---|---|
| 회귀 잔차 산출 | sklearn으로 결정론적 계산 |
| 임대수익 환원율 | 단순 나눗셈 |
| 도보 거리 → 점수 변환 | 임계값 룰 |
| 카드 점수 가중평균 | 단순 가중평균 |

→ *모두 sklearn / numpy / Python으로 룰 계산*. LLM은 숫자 처리 약함.

### 3.2. 가중치 산출

| 작업 | 사유 |
|---|---|
| 사용자 답 → sub-item 가중치 (1.0/0.5/0.0) | 룰 매트릭스 |
| 답 vector → 카드 내 비중 분배 | 정규화 함수 |
| 페르소나 식별 | 우리는 페르소나 안 씀 (manifesto §3) |

### 3.3. 트리거 판정

| 작업 | 사유 |
|---|---|
| 미세먼지 국지 초과 ≥ 50%? | 임계값 비교 |
| 잔여 프리미엄 ≥ 20%? | 임계값 비교 |
| 도로 거리 ≤ 200m? | 임계값 비교 |

### 3.4. 추론·예언

| 작업 | 사유 |
|---|---|
| 5~10년 가격 예측 | manifesto §우리가 약속하는 것 — 단일 가격 예측 안 함 |
| 호재 실현 가능성 | manifesto §우리가 약속하는 것 — 호재 예언 안 함 |
| 인과 추론 ("학교 가까우니 미래가치 ↑") | hallucination 위험 큼 |
| 시장 전망 | 우리 범위 밖 |

### 3.5. 자유 형식 발언 생성

| 작업 | 사유 |
|---|---|
| "옹호 agent 발언을 자유롭게 만드세요" | 출처·사실 통제 불가 |
| "이 매물에 대한 종합 의견 1~2 단락" | 외삽·할루시네이션 위험 |
| "사용자 답에 대한 자유 응답" | 비결정론적 |

→ 토론 발언은 **사전 템플릿 + 플레이스홀더 채우기**만.

### 3.6. 사실에 없는 정보 추론

| 작업 | 사유 |
|---|---|
| RAG에 없는 학원 평판 추론 | 외삽 |
| 데이터에 없는 시공사 평판 | 외삽 |
| 단지 분위기·평판 | 데이터 없음 |

## 4. 안전 패턴 — 3가지

### 패턴 A: 분류 (Classification)

```python
def llm_classify(input_data, options):
    """LLM에게 사전 정의 옵션 중 선택만 시킴"""
    prompt = f"""다음 데이터를 보고 옵션 중 하나만 선택하세요.
    
    데이터: {input_data}
    옵션: {options}
    
    답변 형식: 옵션 중 하나만 출력 (다른 말 없이)"""
    
    output = llm.complete(prompt, temperature=0)
    
    # 검증
    if output.strip() not in options:
        return fallback_rule_classify(input_data)
    return output.strip()
```

### 패턴 B: 플레이스홀더 채우기 (Template Fill)

```python
def llm_fill_template(template, slots, data):
    """템플릿의 슬롯만 LLM이 채움"""
    prompt = f"""다음 슬롯만 채우세요. 
    템플릿: {template}
    데이터: {data}
    슬롯: {slots}
    
    제약: 슬롯 외 변경 금지, 모든 숫자·고유명사 데이터에서 그대로"""
    
    output = llm.complete(prompt, temperature=0)
    
    # 검증
    if not validate_template_structure(output, template):
        return fallback_rule_fill(template, slots, data)
    return output
```

### 패턴 C: 자연어 다듬기 (Polish)

```python
def llm_polish(rule_text, key_facts):
    """룰로 생성된 텍스트를 자연스럽게 정리. 사실 변경 X."""
    prompt = f"""다음 문장을 더 자연스럽게 다듬되, 모든 숫자·고유명사 그대로.
    원본: {rule_text}
    핵심 사실: {key_facts}"""
    
    output = llm.complete(prompt, temperature=0)
    
    # 검증
    for fact in key_facts:
        if fact not in output:
            return rule_text  # 다듬기 실패, 원본 사용
    return output
```

## 5. RAG 사용 시 주의사항

### 5.1. 컨텍스트 명확히 제공

```
✅ 좋은 prompt:
"다음 *데이터만* 사용해 답하세요:
- 단지: 광명 OO아파트
- 회귀 잔차: +18%
- 학교 거리: 도보 5분
컨텍스트에 없는 정보는 '데이터 부족'으로 답하세요."

❌ 나쁜 prompt:
"이 매물에 대해 의견을 주세요."
```

### 5.2. 컨텍스트 외 정보 차단

prompt 마지막에 *명시적 제약*:
```
"위 데이터에 없는 사실은 답에 포함하지 마세요.
'알려져 있다', '~할 수 있다' 같은 추측 표현 금지."
```

### 5.3. 출처 표시 강제

```
"숫자 인용 시 [출처] 표기. 예: 잔차 +18% [국토부 실거래가]"
```

## 6. 우리 5장 카드 — LLM 사용 매트릭스

각 카드의 LLM 사용 영역 명시:

| 카드 | 점수 산출 | 자연어 다듬기 | 분류 | 결정 가이드 |
|---|---|---|---|---|
| 시세 | 룰 (회귀+환원율) | LLM ✅ (옹호·반대 발언 다듬기) | LLM ✅ (잔차 → 거품 강도) | — |
| 입지 | 룰 (도보 거리) | LLM ✅ | — | — |
| 생활편의 | 룰 (인프라 카운트) | LLM ✅ | — | — |
| 미래가치 | 룰 (회귀+이벤트 스터디) | LLM ✅ | LLM ✅ (호재 의존도 → 강도) | — |
| 리스크 | 룰 (노후도+환경) | LLM ✅ | LLM ✅ (위험 등급) | — |
| 종합 | 룰 (가중평균) | — | — | LLM ✅ (한 줄 가이드) |

→ 핵심은 **점수 산출 0% LLM**. 부가 자연어만 LLM.

## 7. 가중치 토론에서의 LLM 사용

가중치 토론(인터뷰 → 답 vector → 비중 분배 → 토론 화면)의 각 단계:

| 단계 | LLM? | 사유 |
|---|---|---|
| 인터뷰 질문 *생성* | ❌ | 사전 정의 템플릿 + 매물 데이터 자동 삽입 |
| 사용자 답 → sub-item 가중치 (1.0/0.5/0.0) | ❌ | 룰 매트릭스 |
| 답 vector → 카드 내 비중 분배 | ❌ | 정규화 함수 |
| 옹호·반대·중립 *발언 다듬기* | ✅ | 패턴 C (자연어 다듬기) |
| 사용자에게 추가 질문 *선택* (충돌 분기) | ❌ | 사전 정의 결정 트리 |

→ 토론 *발언 자체*는 룰로 생성. LLM은 *문장 자연스러움*만.

## 8. 검증 절차

### 8.1. 자동 검증 (런타임)

```python
def validate_llm_output(output, validation_rules):
    """LLM 출력 자동 검증"""
    checks = [
        check_length(output, max_len=200),
        check_options(output, allowed_options),
        check_facts_present(output, required_facts),
        check_no_hallucination_keywords(output, blocklist=['알려져 있다', '~할 수 있다']),
    ]
    return all(checks)
```

### 8.2. Fallback 룰

LLM 출력이 검증 실패 시 *룰로 생성한 결과 사용*:

```python
output = llm_classify(...)
if not validate(output):
    output = rule_based_classify(...)  # fallback
```

→ **시연 시 LLM 실패해도 시스템 작동** 보장.

### 8.3. 사전 검증 (시연 전)

- 동일 케이스 10회 실행 → 출력 일관성 확인
- 엣지 케이스 (극단값) 통과 확인
- 검증 함수 모든 출력에 적용
