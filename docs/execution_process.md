# Execution Process

부동산 검증 AI 에이전트 — 런타임 흐름·오케스트레이션

기준일: 2026-05-05

---

## 한 줄 요약

> **매물 주소 입력 → 식별 → 기본 인터뷰 → 5장 카드 *병렬* 분석 → 가중치 인터뷰 → 3 에이전트 토론 → 사용자 가중치 조정 → 최종 점수.** 7단계 흐름, 모든 단계 *결정론적*(LLM 의존 0%).

---

## 2. 단계별 상세

### Stage 1 — 매물 식별

**목적**: 사용자 자유 형식 주소 → 표준 매물 키 (`HSMP_INNB`)

**흐름**:
```
1. 사용자 입력 (자유 형식)
   예: "광명시 OO아파트 105동 1203호"
   ↓
2. juso.go.kr 주소 텍스트 검색 (#30)
   → 도로명주소 + 지번주소 + 우편번호 + 행정동 표준화
   ↓
3. vworld 지오코딩 (#32) — juso 좌표(#31) 심사 중이므로 vworld가 1차
   → 위경도 (EPSG:4326)
   ↓
4. 한국부동산원 단지 연계정보 (#39, RAG)
   → HSMP_INNB + PNU + 법정동코드 매핑
   ↓
5. 단지 컨펌
   → 지도에 핀 표시 + 사용자에게 "이 매물이 맞습니까?" 확인
```

**예외 처리**:
- 주소 매칭 실패 → 유사 단지 후보 3개 제시 → 사용자 선택
- 단지 매칭 실패 (HSMP_INNB 없음) → "지원하지 않는 매물 형식" 안내 (오피스텔 단독 매물 등 일부)

**데이터 호출** (4건):
- juso 검색 (1회)
- vworld 지오코딩 (1회)
- 한국부동산원 단지 연계정보 (1회 — 캐시 가능, 단지 단위 데이터)
- 단지 기본정보 #8 (단지명·세대수 표시용)

**소요 시간**: ~2초

---

### Stage 2 — 기본 인터뷰

**목적**: 분석 시작 전 *최소한의* 사용자 컨텍스트 수집

**질문**:

| Q | 질문 | 사용 |
|---|---|---|
| Q1 | 회사 위치 + 출퇴근 수단 (선택) | 입지 카드 출퇴근 시간 산출 (ODsay #37) |
| Q2 | 가족 구성 (선택) | 평형 적합성 + 학군 가중치 보정 |
| Q3 | 우선순위 1~2개 (필수) | 자산가치 / 출퇴근 / 거주환경 / 학군 → 카드 표시 순서 결정 |

**원칙**:
- *모두 선택*. 답변 없어도 분석 가능 (균등 가중치 적용)
- 자산·재무 정보 *절대 안 받음* (`manifesto.md` §3.4)

**소요 시간**: ~1분

---

### Stage 3 — 5장 카드 병렬 분석

**목적**: 5개 도메인 객관 점수 산출

**병렬 처리**:
```python
# 5개 카드 비동기 병렬 호출
results = await asyncio.gather(
    analyze_price(hsmp_innb, area_code, coords),
    analyze_location(hsmp_innb, coords, user_office),
    analyze_lifestyle(hsmp_innb, coords),
    analyze_future(hsmp_innb, area_code, coords),
    analyze_risk(hsmp_innb, coords),
)
```

#### 3.1. Card 1 — 시세 분석 흐름

```
1. HSMP_INNB → 행정동코드·평형·연식 식별 (단지 연계정보 #39)
   ↓
2. 인근 단지 패널 데이터 호출
   - 같은 시군구 + 비슷한 평형(±5평) + 비슷한 연식(±10년)
   - 최근 12개월 실거래가 (#1, #3, #5)
   - 최소 30~50개 거래 데이터 확보
   ↓
3. 회귀 모델 적합 (sklearn.linear_model.LinearRegression)
   평당가 ~ 연식 + 세대수 + 평형 + 역세권 거리 + 학교 거리
   ↓
4. 회귀 적정가 산출 + 잔차 = 실제 평당가 - 회귀 적정가
   ↓
5. 임대수익 환원율 보정 (#2 전월세 데이터)
   환원율 = 연 임대수익 ÷ 매매가
   ↓
6. 단지별 직거래 비율 검사 (#52)
   직거래 비율 > 30% → 트리거 (정상 시세 왜곡 가능성)
   ↓
7. 별점 산출
   잔차 < 5%, 환원율 시장 평균 ±10% → ★5.0
   잔차 5~10%                       → ★4.0
   잔차 10~20%                      → ★3.0
   잔차 20~30%                      → ★2.0
   잔차 30%+                        → ★1.0
```

**데이터 호출**: #1, #2, #3, #5, #35, #39, #52 (7건)

2. 핵심 인프라 거리·개수 집계

#### 3.4. Card 4 — 미래 가치 흐름

```
1. HSMP_INNB → 시세 카드의 회귀 잔차·이벤트 스터디 결과 재사용
   ↓
2. 호재 의존도 계산
   = (실제 평당가 - 회귀 적정가) ÷ 회귀 적정가 × 100%
   ↓
3. 인근 정비사업 단계 조회 (서울시 #46 — 한국부동산원 RAG)
   - 단계별 평균 소요년수 baseline (추진위→입주 평균 13년)
   ↓
4. 신규 공급 추정 (#25, #26)
   - 향후 3년 인근 분양 물량
   - 청약 경쟁률 → 매수 압력 시그널
   ↓
5. 인구 추세 검사 (#20, #22, #50)
   - 행정구 30대 인구 5년 변화
   - 학령인구 변화
   - 배정 초등 입학생 추세
   ↓
6. 거시 환경 보정 (#35 ECOS)
   - 기준금리·CPI·가계대출
   ↓
7. 5~10년 시나리오 분기 (호재 0/50/100% 가정)
   ↓
8. 별점 산출
   호재 의존도 < 5% + 인구 유입 + 정비사업 진행 → ★5.0
   호재 의존도 < 15%                              → ★4.0
   호재 의존도 < 25%                              → ★3.0
   호재 의존도 < 35%                              → ★2.0
   호재 의존도 ≥ 35%                              → ★1.0
```

**데이터 호출**: #25, #26, #46, #50, #47, #52, #20, #22, #35 (9건)

#### 3.5. Card 5 — 리스크 (거주 환경) 흐름

```
1. HSMP_INNB → 건축물대장 #7 + 공동주택 기본정보 #8
   - 사용승인일·구조·세대수·주차대수
   ↓
2. 단지 노후도·관리 품질 점수
   ↓
3. 미세먼지 국지 초과 비율 산출
   - 가장 가까운 측정소 매칭 (#23 측정소 정보)
   - 1년치 일평균 PM2.5 시계열
   - 시 평균 동기 데이터
   - 초과 일수 비율 = (인근 측정소 > 시 평균) ÷ 365
   ↓
4. 인근 도로교통 대기오염 검사 (#54 한국부동산원 RAG)
   - CO₂·NOx·VOC·미세먼지 일평균
   ↓
5. 교통 소음 프록시 (#10·#11·#14 + 단지 좌표)
   - 가장 가까운 간선도로 거리
   ↓
6. 표고·경사도 (#53, #49 한국부동산원 RAG)
   - 침수 리스크 (저지대 < 10m)
   - 보행 부담 (경사도 > 20°)
   ↓
7. 환금성 시그널 (#45 부동산중개업 현황)
   - 500m 내 중개업소 수·영업률
   ↓
8. 별점 산출
   노후 < 10년 + 환경 양호 + 환금성 양호 → ★5.0
   노후 < 20년 + 국지 초과 < 30%          → ★4.0
   노후 < 30년 + 국지 초과 < 50%          → ★3.0
   노후 30년+ 또는 국지 초과 50%+         → ★2.0
   노후 + 환경 + 환금성 모두 부적합        → ★1.0
```

**데이터 호출**: #7, #8, #23, #54, #53, #49, #45, #10, #11, #14 (10건)

**트리거 조건**:
- 미세먼지 국지 초과 ≥ 50% → 발생원 매핑 (54번 + 16번 인근 공장 분석)
- 간선도로 ≤ 200m → 도로 거리·방향 표시
- 표고 < 10m → 침수 안내문

**Stage 3 소요 시간**: ~30초~1분 (병렬 + 캐싱 적용 시)

### Stage 4 — 가중치 결정 인터뷰

**목적**: 사용자 가치관에 따른 자동 초기 가중치 산출

**5문항**:

| Q | 질문 | 답변 옵션 | 영향 카드 |
|---|---|---|---|
| Q1 | 보유 기간? | 5년 이상 / 단기 (3년 이하) | 시세·미래가치·리스크 |
| Q2 | 자녀 학군 중요? | 매우 중요 / 보통 / 비해당 | 입지 |
| Q3 | 출퇴근 vs 거주환경 | 출퇴근 우선 / 거주환경 우선 / 균형 | 입지·생활편의·리스크 |
| Q4 | 호재 신뢰도? | 실현 신뢰 / 회의적 / 중립 | 시세·미래가치 |
| Q5 | 자산 가치 vs 거주 만족 | 자산 / 거주 / 균형 | 전 5장 |

**룰 매트릭스 적용** (인터뷰 답변 → 가중치 변동량):

```
초기 가중치: 시세 20% / 입지 20% / 생활편의 20% / 미래가치 20% / 리스크 20%

Q1 = "5년 이상"  → 미래가치 +5%, 리스크 -3%
Q1 = "단기"      → 시세 +5%, 미래가치 -5%

Q2 = "매우 중요" → 입지 +10% (학교 부분에 가중)
Q2 = "보통"      → 변동 없음

Q3 = "출퇴근"    → 입지 +5%
Q3 = "거주환경"  → 생활편의 +3%, 리스크 +3%

Q4 = "실현 신뢰" → 미래가치 +10%
Q4 = "회의적"    → 시세 +5%, 미래가치 -10%

Q5 = "자산"     → 시세 +3%, 미래가치 +3%
Q5 = "거주"     → 생활편의 +3%, 리스크 +3%
```

→ 변동 후 정규화 (합 = 100%)

**소요 시간**: ~1~2분

### Stage 5 — 3 에이전트 가중치 토론

**목적**: 사용자 답변에 따른 *3가지 관점*의 가중치 추천 시각화

**3 에이전트 룰**:

```
🟢 옹호 에이전트
  - 매물의 *긍정 측면*을 강조하는 가중치 추천
  - 룰: 점수 높은 카드(★4.0+)에 가중치 +5%
       점수 낮은 카드(★2.5-)에 가중치 -3%
  - 메시지 예: "보유 5년+이라면 미래가치 가중치 ↑"

🔴 반대 에이전트
  - 매물의 *부정 측면*을 강조하는 가중치 추천
  - 룰: 점수 낮은 카드(★2.5-)에 가중치 +5%
       점수 높은 카드(★4.0+)에 가중치 -3%
  - 메시지 예: "노후 25년 + 도로 인접 → 리스크 가중치 ↑"

⚪ 중립 에이전트
  - 균형 + 사용자 자율 강조
  - 룰: Stage 4 인터뷰 룰 결과 그대로
  - 메시지 예: "두 의견 다 일리. 사용자 직접 가중치 조정 권장"
```

**각 에이전트가 산출하는 결과**:
- 추천 가중치 (5개 카드 합 100%)
- 그에 따른 종합 점수 (가중평균)
- 한 줄 메시지

**예시** (매물 점수 시세 3.2 / 입지 4.5 / 생활편의 3.8 / 미래가치 4.0 / 리스크 2.8):

```
🟢 옹호: 미래가치 30% / 입지 25% / 생활편의 20% / 시세 15% / 리스크 10%
       → 종합 ★3.85
       
🔴 반대: 리스크 30% / 시세 25% / 생활편의 20% / 입지 15% / 미래가치 10%
       → 종합 ★3.50
       
⚪ 중립: 인터뷰 룰 적용 결과
       → 종합 ★3.66
```

**LLM 사용 0%** — 모든 룰은 데이터·답변에서 자동 도출.

**소요 시간**: ~1초 (계산만)

---

### Stage 6 — 사용자 가중치 조정 (인터랙티브)

**목적**: 사용자가 *자기 우선순위를 자각*하면서 가중치 직접 조정

**UI**:
- 5개 카드별 슬라이더 (또는 별점 1~5)
- 합 100% 자동 정규화
- 종합 점수 실시간 재계산

**상호작용**:
```
사용자 슬라이더 변경
  ↓
종합 점수 = Σ (카드 점수 × 사용자 가중치) / Σ 가중치
  ↓
화면 즉시 갱신 (밀리초 단위)
```

**효과**:
- 사용자가 *"미래가치 강조하면 점수가 어떻게 변하지?"* 직접 실험
- 페르소나 약속 *"사용자를 더 똑똑하게"* 의 정확한 구현체

**소요 시간**: 사용자가 만족할 때까지 (보통 1~3분)

├────────────────────────────────────────────────────┤

│  시세 15% · 입지 25% · 생활편의 20% ·               │
│  미래가치 30% · 리스크 10%                          │
├────────────────────────────────────────────────────┤
│  💬 결정 가이드                                     │

## 3. 데이터 호출 패턴

### 3.1. 캐싱 전략

| 데이터 유형 | 캐싱 기간 | 사유 |
|---|---|---|
| 단지 마스터 (39, 8, 9) | 24시간 | 거의 안 변함 |
| 실거래가 (1~6) | 24시간 | 월 갱신이지만 최신 호출 비용 줄임 |
| 인구·세대 (20~22) | 24시간 | 월 갱신 |
| 도시공원·학교·병원 등 (18, 24, 33, 34) | 7일 | 분기·연 갱신 |
| 정비사업·청약 (#46, #25, #26) | 7일 | 월·수시 갱신 |
| 에어코리아 (#23) | 1시간 | 시간별 측정 |
| 좌표 변환 (#30, #31, #32) | 영구 | 주소→좌표 매핑 안 변함 |
| ODsay 경로 (#37) | 1시간 | 사용자 입력 좌표 조합별 |

### 3.2. 비동기 병렬

```python
# Stage 3 — 5장 카드 병렬
results = await asyncio.gather(*card_tasks)

# 카드 내부에서도 데이터 호출 병렬
async def analyze_price(hsmp_innb, ...):
    txn_data, rent_data, debug_data = await asyncio.gather(
        fetch_transactions(hsmp_innb),
        fetch_rentals(hsmp_innb),
        fetch_direct_trades(hsmp_innb),
    )
    ...
```

### 3.3. 에러 처리

| 에러 유형 | 처리 |
|---|---|
| API 호출 실패 (네트워크) | 3회 지수 백오프 재시도 → 실패 시 *해당 데이터 없음*으로 분석 진행 |
| 데이터 부족 (인근 거래 < 10건) | 해당 카드 점수 산출 불가 → "데이터 부족" 표시 + 별점 미표시 |
| 단지 매칭 실패 | "지원하지 않는 매물" 안내 + 종료 |
| 좌표 변환 실패 | juso → vworld 백업 → 둘 다 실패 시 사용자 재입력 요청 |
| 측정소 부재 (#23 매물 인근 측정소 없음) | 시·구 단위 평균으로 fallback + 신뢰도 ↓ 표시 |

### 3.4. 데이터 호출 횟수 (시연 매물 1건당)

| Stage | API 호출 | 캐싱 활용 |
|---|---|---|
| 1. 매물 식별 | 4건 | 단지 매스터 캐싱 |
| 3. 5장 카드 분석 | 약 30~40건 (병렬) | 70% 캐싱 적용 시 실제 호출 ~12건 |
| 5. 토론 (계산만) | 0건 | — |
| **합계** | **~16건 호출** | (캐싱 후) |

| 단계 | 목표 시간 |

## 6. 시연 시 흐름 (실제 시간)

```
0:00  사용자: 매물 주소 입력
      "광명시 OO아파트 105동 1203호"
0:03  시스템: 표준화 + 좌표 변환 + HSMP_INNB 매핑
      → 지도에 핀 표시 + "이 매물이 맞습니까?" 컨펌

0:10  사용자: 컨펌 → 기본 인터뷰 시작
      Q1 회사 위치 / Q2 가족 / Q3 우선순위
      
1:00  사용자: 인터뷰 완료 → 분석 시작 버튼

1:05  시스템: "5명의 전문가가 분석 중입니다…"
      백그라운드 5장 카드 병렬 분석

1:35  시스템: 분석 완료 → 별점 카드 5장 표시

1:40  사용자: 가중치 결정 인터뷰 시작 (Q1~Q5)

3:00  사용자: 인터뷰 완료
      
3:01  시스템: 3 에이전트 토론 시각화
      옹호/반대/중립 가중치 추천

3:30  사용자: 가중치 슬라이더 직접 조정
      종합 점수 실시간 변화

5:00  사용자: 만족 → 최종 점수 + 결정 가이드 확인

5:00  완료
```

→ **총 5분 시연**.

## 7. 의사결정 이력

이 실행 프로세스의 주요 설계 결정:

| 결정 | 사유 |
|---|---|
| **5장 카드 비동기 병렬** | 직렬 호출 시 5분+, 병렬로 1분 이내 |
| **회귀 분석에 LLM 사용 안 함** | sklearn 결정론적 모델 (`manifesto.md` §1.5) |
| **3 에이전트 룰 기반** | LLM 토론 생성 안 함 (할루시네이션 회피) |
| **사용자 가중치 슬라이더 실시간** | 종합 점수 즉시 재계산 (캐시 데이터로) |
| **"30분 소요" 표기는 시연용 과장** | 실제 1분 이내 — 사용자 기대 관리 |
| **에러 시 graceful degradation** | 데이터 일부 없어도 분석 진행 (해당 카드 신뢰도 표시) |
