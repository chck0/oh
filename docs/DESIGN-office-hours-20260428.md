## Problem Statement

한국 30대 직장인은 수억 원짜리 부동산 투자 결정을 내릴 때 접근 가능하고 신뢰할 수 있는 자문을 받기 어렵다.
전문가 자문은 수백만 원에 이해충돌 문제(중개인은 거래를 성사시키고 싶어한다)가 있고,
DIY 대안은 데이터는 있지만 해석 능력이 없다.

---

## Demand Evidence

| 항목 | 내용 |
|------|------|
| 창업자 본인 경험 | 아파트 매수 시 "좋은 거인지" 판단 불가 → 전문가 자문 시도 → 너무 비싸거나 접근 불가 |
| 제3자 검증 | **없음** — 전적으로 창업자 본인 경험에 의존 |
| 위험 | 수요 가설이 단일 일화에 기반. PMF 확인 전 기능 추가는 의미 없음 |

**최우선 과제: 검증.** 이 설계 문서의 목적은 제품을 완성하는 것이 아니라
진짜 수요가 있는지 확인하는 가장 빠른 경로를 설계하는 것이다.

## Status Quo (현재 대안)

사용자들이 지금 실제로 하는 것:

1. **네이버 카페 / 유튜브** — 무료이지만 콘텐츠 제작자가 본인 이해관계로 편향
2. **부동산 컨설턴트 / 중개인** — 이해충돌, 수백만 원 비용
3. **호가 앱 (직방, 다방 등)** — 데이터는 있지만 해석은 본인이 해야 함

갭: **실거래 데이터 + 전문 해석 + 저렴한 비용**을 동시에 제공하는 솔루션 없음.

## Target User & Narrowest Wedge

**타겟 유저**: 30대 직장인, 처음 아파트 매수를 고려 중, 데이터는 볼 수 있지만
해석은 어려운 사람. (창업자 본인이 이 유저다.)

**가장 좁은 쐐기 (Narrowest Wedge)**:
- 단일 물건 분석 — 주소 또는 단지명 입력
- 1페이지 판정 출력: 수익률 요약 + 위험 3개 + 종합 판단
- 카카오톡 공유 버튼 (배우자/친구와 공유)
- 가격: 무료 또는 9,900원 이하로 수요 테스트

## Constraints

- 코드베이스 이미 존재 (Python, Streamlit, Claude API, 국토부 실거래가 API)
- 260+ 테스트 통과
- 현재 로컬 전용 실행 — 외부 배포 없음, 외부 사용자 없음, 결제 시스템 없음
- KAIST MBA 데모 맥락은 **별개 유스케이스**로 취급. 이 문서는 소매 투자자(30대 직장인) 제품을 다룬다. MBA 데모 기능(전문가용 PDF 리포트 등)은 이 검증 단계의 범위 밖.
- Approach A에서 실제 결제는 구현하지 않는다 — 인터뷰에서 "9,900원 낼 의향" 언어적 확인만 한다.

## Premises

1. ✅ **한국 소매 투자자(30~40대 직장인)가 부동산 투자 결정을 내릴 때 현재 대안으로는 
   객관적이고 저렴한 자문을 받기 어렵다.** (동의)

2. ✅ **(수정)** "AI가 다양한 관점을 토론하되, 명확한 합성/판정도 함께 제공되어야" 
   신뢰 문제를 해결하면서 사용성도 유지한다.  
   → 토론만 보여주면 첫 구매자는 오히려 더 혼란스러울 수 있다. (크로스모델 도전으로 수정)

3. ✅ **실거래가 데이터(국토부 API) 기반 분석이 블로그/유튜브보다 신뢰할 수 있고, 
   이게 핵심 차별점이다.** (동의)

4. ✅ **이 제품의 첫 번째 유저는 창업자 본인과 비슷한 사람이다 — 
   데이터는 볼 수 있지만 해석이 안 되는 30대 직장인.** (동의)

## Approaches Considered

### Approach A: 1페이지 판정 앱 ✅ (선택)

```
Summary: 기존 Streamlit UI를 단순화. 주소 입력 → 1페이지 판정 출력
         (수익률 + 위험 3개 + 합성 판단) + 카카오톡 공유 버튼.
Effort:  S (1주일 이내)
Risk:    Low
Pros:    - 기존 백엔드 100% 재사용
         - 가장 빠른 수요 검증 경로
         - 카카오톡 공유 = 자연스러운 바이럴 루프
Cons:    - UI 단순화가 기존 기능의 가치를 숨길 수 있음
         - Streamlit 한계 (사용자 인증, 결제 미지원)
Reuses:  src/meeting.py, src/yield_analyzer.py, src/real_estate.py 전체
```

### Approach B: 카카오톡 챗봇

```
Summary: 앱 없이 카카오톡에서 예매 링크 공유 → AI 분석 반환.
Effort:  M (2~3주)
Risk:    Medium
Pros:    - 가입/설치 없음 → 마찰 최소
         - 타겟 유저가 이미 카카오톡에 있음
Cons:    - 카카오 API 학습 및 배포 추가 작업
         - 기존 Streamlit UI 재사용 어려움
Reuses:  src/meeting.py, agents/*.md
```

### Approach C: SaaS 리포트 서비스

```
Summary: 건당 2,900원 또는 월 9,900원 구독. 웹앱 + 로그인 + 결제.
Effort:  XL (4주+)
Risk:    High
Pros:    - 수익화 경로가 명확
         - 완전한 기능 세트 활용 가능
Cons:    - 수요 검증 없이 과투자 위험
         - 인증/결제 인프라 새로 구축 필요
Reuses:  전체 src/ 모듈
```

## Recommended Approach

**Approach A** — 기존 코드 최대 활용, 최소 작업으로 실제 유저 앞에 제품을 놓는다.
수요 검증이 먼저, 기능 완성은 나중.

핵심 변경 사항:
1. Streamlit UI에 "1페이지 요약 판정" 뷰 추가 (토론 내용은 접을 수 있게 expander 처리)
2. **1페이지 판정 출력 필드 (구체 스펙)**:
   - 표면 수익률 (연 임대 수익 / 투자금, %) — `src/yield_analyzer.py` 기준
   - 실질 수익률 (세후, 관리비 차감) — 동일 모듈
   - 시장 평균 대비 상대 순위 (해당 구 평균과 비교)
   - 위험 항목 상위 3개 (CFO·CSO·컨설턴트 의견에서 합성, 각 1문장 이내)
   - **합성 판단 레이블**:
     - 🟢 매력적: 표면 수익률 4% 이상 AND 위험 항목 1개 이하
     - 🟡 보통: 표면 수익률 2~4% OR 위험 항목 2개
     - 🔴 위험: 표면 수익률 2% 미만 OR 위험 항목 3개 이상
   - 면책 고지: "본 분석은 참고용입니다. 투자 결정은 본인 책임입니다."
3. 카카오톡 공유: JS SDK 방식 대신 **카카오 공유 URL 스킴** (`https://sharer.kakao.com/talk/friends/picker/link`) 사용 — Streamlit `st.link_button()`으로 구현, 별도 JS 삽입 불필요
   - 선결 조건: Streamlit Community Cloud 배포 도메인 확정 후 카카오 개발자 앱 등록 필요 (약 1~2일 추가)
4. "상세 분석 보기" 토글 — 기존 CFO/CSO/컨설턴트 전체 토론 내용을 expander로 접어서 제공

## Open Questions

1. **수요 실체**: 창업자 외에 실제로 이 제품에 9,900원을 낼 사람이 있는가?

## The Assignment

**지금 당장 해야 할 한 가지:**

네이버 '부동산 스터디' 카페나 직장인 커뮤니티에서 **최근 3개월 내 아파트 매수를 진지하게 고려하거나 진행한 사람 5명을 찾아서 30분 인터뷰하세요.**

인터뷰에서 확인해야 할 것 (이 순서로):
1. "그 결정을 내릴 때 가장 어려운 점은 무엇이었나요?" — 문제 실체 확인
2. "결국 어떻게 결정하셨나요? 뭘 보고/누구한테 물어보셨나요?" — 현재 대안 확인
3. "지금 이 분석을 보여드리면 9,900원 내실 의향이 있으신가요?" (목업 지참) — 수요 확인

5명 중 2명이 "내겠다"고 하면 → Approach A 즉시 구현.  
5명이 모두 "유튜브로 충분했어요"라고 하면 → 문제 재정의 필요.

기능을 더 만들기 전에 이 인터뷰를 완료하세요.

## What I noticed about how you think

- **전제를 방어하지 않았습니다.** 서브에이전트가 전제 #2를 도전했을 때, 옳다면 인정하는 길을 선택했습니다. "토론 + 합성"이 맞다고 스스로 결론 내렸습니다. 이건 확증 편향을 거부하는 자세입니다.

- **"내 자신 이야기다"라고 했습니다.** 포장하지 않고 솔직하게 말했습니다. 가장 설득력 있는 창업자 이야기는 항상 본인 경험에서 시작합니다.

- **이미 만들어놨습니다.** 아직 아무도 검증하지 않은 상태에서 260+ 테스트가 통과되는 시스템을 구축했습니다. 실행력은 있습니다. 지금 부족한 건 검증입니다.

- **접근방식 A를 선택했습니다.** C(완전한 SaaS)가 아니라 A(빠른 검증)를 골랐습니다. 욕심보다 현실을 선택한 겁니다. 좋은 신호입니다.
