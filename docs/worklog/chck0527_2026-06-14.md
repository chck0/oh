## 2026-06-14 — OSM 지하철 선형 데이터 연동 + 폴리라인 렌더링 완성

**의도:** 카카오맵 위에 지하철 경로를 역-역 직선이 아닌 실제 선로 선형(OSM Overpass 데이터)으로 표시.

**최종 상태:** 완료.
- OSM Overpass API로 수도권 지하철 251개 노선 선형 다운로드 → `data/subway_shapes_kr.json` (11.5MB)
- `app/subway_shapes.py` 신규 — 노선명 정규화 + nearest-point 구간 추출 + `enrich_linestring()`
- `app/detail.py` — `/api/apt/{seq}/routes` 응답 시 지하철 linestring을 OSM 고품질로 교체
- `web/result.html` — route-overlay 투명화(지도 클릭 차단 제거), `closeRoute()`에서 `clearPolylines()` 제거(닫아도 선 유지), `showDetail()`에서 폴리라인 자동 그리기, `strokeStyle` `'dashed'`→`'shortdash'` 수정
- 실 API 테스트: 수도권 2호선 서초→삼성 4pts → 10pts (OSM enrich 확인)

**건드린 파일·함수:**
- `app/subway_shapes.py` (신규): `enrich_linestring()`, `get_segment()`, `_norm()`, `_load_index()`
- `app/detail.py`: routes 응답 빌드 부분 — `enrich_linestring()` 호출
- `web/result.html`: `closeRoute()`, `showDetail()`, `.route-overlay` CSS, `drawRoutePolylines()`
- `scripts/download_subway_shapes.py` (신규): Overpass API 쿼리 + 수도권 필터
- `data/subway_shapes_kr.json` (신규): 11.5MB, 251개 노선

**결정 / 버린 대안:**
- ODsay passShape 대신 OSM: ODsay 무료 플랜은 passShape 제공 안 함
- GTFS shapes.txt 대신 OSM: KTDB는 담당자 신청 방식, OSM은 즉시 무료 사용 가능
- 버스 선형은 현재 방식(정류장 좌표 직선) 유지: 버스 노선 데이터는 별도 API 필요, 범위 밖

**DB 변경:** 없음 (이전 세션에서 migrate_add_linestring.sql 이미 적용)

**함정·깨진 것:**
- `strokeStyle: 'dashed'` → Kakao Maps 미지원, `'shortdash'`로 수정
- `_STRIP_PREFIX`에 `수도권` 단독 미포함으로 "수도권 5호선" 정규화 실패 → 정규식에 추가
- `route-overlay` 배경색이 지도를 가리고 있었음 → `background: transparent; pointer-events: none`
- OSM 타 도시 동명 노선(부산 2호선 등) 혼입 → score(km) 기준으로 자동 필터링됨
