### 함정 / 깨진 것

- **세마포어와 sleep의 미묘한 상호작용**: 재시도 시 `sleep(1s)`을 sem 밖으로 빼야 다른 호출이 슬롯 사용 가능. 안에서 sleep 하면 다른 호출도 1초 멈춤. `_do_call` 분리한 이유.
- **lazy 세마포어 생성**은 처음 호출되는 이벤트 루프에 바인딩됨. FastAPI 한 프로세스 한 루프면 OK지만, 여러 루프에서 호출하는 테스트 코드는 reset 필요 가능. 현재 호출 경로엔 영향 없음.

### 함정 / 깨진 것

- **429 Too Many Requests 폭증** — keep-alive로 셋업 비용이 사라지자 동일 시간 안에 ODsay 키별 RPS 한도를 직격하기 시작. 운영정책(`lab.odsay.com/doc/totalPolicy`)에 "서비스 유형별 초당 API 호출 건수 제한" 명시. 빠르게 보내는 게 항상 좋은 게 아님이 확인됨.
- baseline run 1은 cold start로 결과가 32초까지 튐. Windows 환경에서 첫 aiohttp ClientSession 초기화 + 첫 TLS handshake가 매우 비쌈. 운영 검색 1회차도 같은 문제 가능성 → 서버 워밍업/preconnect 고려 가치.
