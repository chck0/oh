# 작업 일지 (worklog)

조원별로 자기 파일 하나씩 가집니다. **자동으로** 작업 히스토리가 쌓입니다.

조원별 + **커밋한 날짜별**로 파일이 쌓입니다.

```
docs/worklog/
  철희_2026-06-01.md   ← 공유됨 (git에 올라감) — 그날 정리된 작업 일지
  철희_2026-06-03.md   ← 다른 날 커밋 → 다른 파일
  민수_2026-06-02.md
  .author              ← 로컬 전용 (git 제외) — 본인 이름
  .raw/
    철희.md            ← 로컬 전용 — 압축 직전 자동 백업 (요청 ↔ Claude 작업)
    철희-db.md         ← 로컬 전용 — DB 변경 명령/SQL 자동 기록
```

(같은 날 여러 번 커밋하면 그날 파일에 시간별 항목으로 위에 쌓입니다.)

## 왜 쓰나

- 나중에 코드를 합칠 때, 합치는 사람이 **"이 조원이 무슨 의도로 어떻게 바꿨는지"**를
  코드만 보는 것보다 훨씬 빨리 이해합니다.
- 그 코드를 다시 Claude한테 시킬 때, 이 히스토리를 같이 주면 Claude가 **훨씬 더 잘합니다**
  (같은 시도·실수를 반복 안 함).

## 어떻게 동작하나 (자동)

의식적으로 할 일은 거의 없습니다.

1. **처음 한 번** — Claude가 이름을 물어봅니다. 알려주면 `.author`에 저장됩니다.
   그리고 Claude Code가 띄우는 **"훅 허용?" 창을 한 번 승인**하면 끝.
2. **작업 중** — 컨텍스트가 압축될 때마다 요청 흐름·Claude 작업·바뀐 파일이,
   그리고 DB를 바꾸는 명령/SQL을 실행할 때마다 그 내용이 `.raw/`에 자동 백업됩니다
   (여러분은 신경 쓸 것 없음).
3. **"깃에 올려줘"라고 할 때** — Claude가 이번 작업을 `<이름>_<날짜>.md`에 깔끔하게
   정리해서 함께 커밋합니다.

## 규칙

- **각자 자기 파일만** 수정됩니다 → git 충돌이 안 납니다.
- 양식은 신경 쓰지 마세요. Claude가 통일된 형식으로 적습니다.
- 최신 작업이 **맨 위**에 쌓입니다.
- `.author`와 `.raw/`는 **개인용이라 git에 안 올라갑니다.** 올라가는 건 정리된 `<이름>_<날짜>.md`뿐.
