# Manifesto

부동산 검증 AI 에이전트 — 핵심 가치·설계 원칙·약속

기준일: 2026-05-05

---

## 한 줄 요약

> **점찍어둔 아파트 한 채를 *5개 도메인 + 사용자 가치관*으로 검증하는 *2차 의견(Second Opinion)* 도구.**

---

## 우리가 믿는 것 — Core Values

### 1. AI는 결정 대신해주지 않는다

AI는 사용자를 *더 똑똑하게* 만든다. 매수 결정은 사용자가 한다. 시스템은 *분석*만 제공하고, *판단*은 사용자에게 위임한다.

### 2. 데이터에 기반한다 — 영업 멘트가 아니라

부동산 시장은 정보 비대칭이 심하다. 중개인의 영업 멘트, 유튜브의 양극단 주장, 부모님의 추측 — 이 노이즈를 걷어낼 *공공 데이터 기반 객관 검증*만 한다.

### 3. 100% 공공 데이터 — 민간 플랫폼 의존 0%

직방·호갱노노·리파인 같은 민간 부동산 플랫폼 데이터는 사용하지 않는다. **정부·공공기관 OpenAPI와 한국부동산원·공간정보산업진흥원·데이터웨이 RAG 데이터만** 사용한다.

이유: *법적 안정성*, *재현 가능성*, *사용자 신뢰*.

### 4. 개인정보 최소화

사용자 *자산·소득·대출·신용*을 받지 않는다. 시스템은 *매물 객관 검증*만 한다. "내가 살 수 있는가"는 사용자가 직접 판단한다. 시스템은 "이 매물이 적정한가"만 답한다.

### 5. LLM 할루시네이션 0%

모든 카드 점수·가중치 추천·트리거 판정은 *데이터에서 룰로* 산출한다. LLM이 *예언*하거나 *논리를 생성*하지 않는다. 시연 시 매번 일관된 결과.

### 6. SLC

*Simple, Lovable, Complete*. 시연 시점에 **모든 카드가 완결 작동**해야 한다. "검증 후 추가" 같은 *연기형 약속*을 하지 않는다.

들어가는 건 다 작동, 안 들어가는 건 명시적으로 시스템 범위 밖.

### 7. 데이터로 못 잡는 건 아웃풋에 안 담는다

페인포인트 매핑이 풍부해도 데이터 가용성이 약하면 *시스템 범위 밖*으로 명시한다. **약속을 줄여서 약속을 지킨다.**

## 우리가 따르는 것 — Design Principles

### 1. 역산 (Output-First)

아웃풋 이미지부터 그린다. 거기서 필요한 분석 결과를 정의하고, 거기서 필요한 데이터를 검증한다.

> *데이터 못 가져오는 항목은 아웃풋에 못 담는다.*

### 2. 다관점 (Multi-Perspective)

시세·입지·생활편의·미래가치·리스크 — **5개 도메인이 서로 다른 관점**에서 같은 매물을 평가한다. 종합 점수는 *5개 도메인 + 사용자 가치관 가중치*의 결합.

### 3. 개인화 (Personalization)

객관 점수는 *모든 사용자에게 동일*하지만, 종합 점수는 *사용자 가치관에 따라 다름*. 인터뷰 5문항 → 룰 기반 가중치 추천 → 사용자 직접 조정.

### 4. 트리거 기반 적응형 정보 깊이 (Adaptive Depth)

정상 매물엔 *안심 신호 한 줄*, 위험 신호 트리거 시에만 *발생원·상세 분석*. 정보 과잉을 막고, 의미 있는 차이만 드러낸다.

### 5. 공식 발표만 다룬다

도시계획 결정·정비사업 추진 경과·청약 분양 같은 *공식 고시·결정 사항*만 호재로 다룬다. 언론 추측·민간 사업·검토 단계는 시스템 범위 밖.

## 우리가 약속하는 것 — Promises

| 약속 | 의미 |
|---|---|
| **자산·재무 정보 안 받음** | 어떤 개인 금융 정보도 시스템에 입력되지 않음 |
| **매물 호 단위 검증 안 함** | 층간소음·내부 누수·곰팡이는 임장 위임. 시스템은 *단지 단위*까지 |
| **권리관계·법무 자문 안 함** | 등기부등본은 인터넷등기소 700원 직접 발급 |
| **호재 실현 예언 안 함** | "GTX 들어올까"가 아니라 *"현재 가격에 호재 기대치 N% 끼어있다"*만 |
| **5~10년 단일 가격 예측 안 함** | 시나리오 분기(호재 0/50/100%)로 *사용자 신뢰도에 따라 변하는 추정*만 |
| **시연 시점에 모든 약속 지킴** | "추후 검증" 같은 약속이 시안에 안 들어감. 보여주는 건 다 작동 |

## 의사결정 이력 — Major Decisions

이 매니페스토에 도달하기까지의 결정.

### ✅ 받아들임

- 5장 카드 도메인 분류 (시세·입지·생활편의·미래가치·리스크)
- 사용자 가중치 토론 (인터뷰 → 3 에이전트 룰 기반 가중치 추천 → 사용자 조정)
- 트리거 기반 적응형 정보 깊이 (정상엔 안심 신호, 위험에만 깊이)
- 공공 OpenAPI 27 + 별도 API 8 + RAG 20 = **총 55개 데이터**
- 미세먼지 *상대값* (시 평균 대비 국지 초과 비율)
- 호재 의존도 분해 (회귀 잔차 + 이벤트 스터디)
- 5~10년 시나리오 분기 (사용자 신뢰도 슬라이더)
- 단지 마스터 키 `HSMP_INNB` 기반 데이터 join

### ❌ 거부

| 거부 사항 | 사유 |
|---|---|
| 재무 카드 (자산·대출 입력) | 개인정보 민감도, 자산 컨설팅화 |
| 법무 자문 (등기·근저당) | 법적 책임, 사용자 직접 발급 가능 |
| 자연재해 카드 | 서울 도시 매물 의미 약함 (정식 지정 위험지구 99% 미해당) |
| 직방·호갱노노·리파인 데이터 | 민간 플랫폼 의존, 법적·재현성 |
| LLM 토론 논리 생성 | 할루시네이션 위험, 일관성 부족 |
| 매물 호 단위 검증 (층간소음·결함) | 임장에서 본인이 확인할 영역 |
| 호재 실현 예언 | 페르소나 가치(*"AI는 결정 안 함"*)와 정면 충돌 |
| 5~10년 단일 가격 예측 | 사용자 신뢰도에 따라 분기되어야 함 |
| LocalData 통합 시스템 의존 | 폐쇄됨 (개별 표준데이터셋으로 대체) |
| MVP 사고방식 | SLC로 좁혀서 시연 시 완결 작동 |

## 이 매니페스토의 역할

이 문서는 *불변의 원리*. 시스템 구현·디자인·미팅 의사결정에서 *판단 기준*이 된다.

새 결정이 들어올 때 이 매니페스토와 충돌하면, *결정을 다시 검토*하거나 *매니페스토를 명시적으로 갱신*한다. 암묵적 위반은 없다.
