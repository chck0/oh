# BADUGI 매니페스토

## 우리가 만드는 것

부동산을 잘 아는 친구가 옆에 있다면 어떨까.

직장 주소 하나를 넣으면, 친구처럼 솔직하게 말해준다.
"이 동네 지하철 직통이야. 근데 요즘 호가가 좀 올랐어."

분석 리포트가 아니다. 카톡 한 줄이다.

---

## 우리가 해결하는 문제

한국에서 아파트를 고를 때 대부분의 사람은 두 가지 함정에 빠진다.

**함정 1. 정보 과잉**
직방, 호갱노노, 네이버 부동산, 부동산 카페, 유튜브 —
쏟아지는 정보 앞에서 정작 "내가 이 집을 사도 되나?"는 여전히 모른다.

**함정 2. 결정 위임**
중개사, 가족, 커뮤니티 댓글에 결정을 맡긴다.
조언은 많은데 내 조건에 맞는 조언은 없다.

BADUGI는 이 두 함정 사이 어딘가에 서 있다.
정보를 줄이되, 내 상황에 맞게.

---

## 우리가 믿는 것

**1. 통근은 삶의 질이다**
집값보다 출퇴근 시간이 행복에 더 큰 영향을 미친다.
모든 추천의 첫 번째 축은 통근시간이다.

**2. 비교가 판단을 만든다**
"이 집이 좋다"는 말은 의미 없다.
"같은 통근시간 내에서 이 집이 가장 싸다"는 말은 유용하다.

**3. AI는 결정하지 않는다**
추천은 하되, 판단은 사용자 몫이다.
BADUGI는 선택지를 좁혀줄 뿐, 마지막 버튼은 사람이 누른다.

**4. 데이터가 없으면 말하지 않는다**
근거 없는 호재, 확인 안 된 학군 — 말하지 않는다.
공식 데이터로 검증 가능한 것만 아웃풋에 담는다.
웹 검색으로 보완할 때는 반드시 출처를 밝힌다.

**5. 친구는 거짓말하지 않는다**
"확인 필요"를 솔직히 붙인다.
실시간 호가·전세가처럼 모르는 건 모른다고 한다.

---

## 우리가 하지 않는 것

- 자산·대출·DSR 시뮬레이션 → 개인정보 영역
- 층간소음·누수 등 호 단위 결함 → 임장(현장 방문)으로 위임
- 5~10년 단일 가격 예측 → 약속을 줄여서 약속을 지킨다
- 직방·호갱노노 데이터 의존 → 공공데이터와 공식 API만

---

## 우리가 만드는 경험

1. 직장 주소 입력 (10초)
2. 조건 설정 (30초) — 최대 통근시간, 예산, 평형
3. 추천 카드 확인 (2분) — 지도 + 친구 한마디
4. 상세 비교 (2분) — 시세 추이, 경로 옵션, 실거래, 도보 시설
5. 친구 채팅으로 깊은 질문 (무제한) — 이미지·문서 첨부, 웹 검색 포함

총 5분. 그 이후는 직접 발로 뛴다.

---

## 친구 채팅이 제공하는 것

spec-22~27을 거쳐 구현된 친구 채팅은 단순 Q&A를 넘어선다:

- **실거래 기반 시세 분석**: 1년 통계·6개월 변동률·동 평균 비교
- **도보 환경 참조**: DB의 도보 10분 이내 시설(학교·마트·역)을 답변에 반영
- **실시간 웹 검색**: 학군·개발호재·정책 질문 시 DuckDuckGo 검색 후 출처 명시
- **이미지·PDF 분석**: 분양 팸플릿 사진, 계약서 PDF를 첨부하면 Claude가 직접 분석
- **대화 유지**: 새로고침 후에도 24시간 이전 대화 복원 (localStorage)
