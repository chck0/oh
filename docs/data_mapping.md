# Data Mapping

부동산 검증 AI 에이전트 — 55개 데이터 × 5장 카드 매핑

기준일: 2026-05-05

---

## 한 줄 요약

> **공공 OpenAPI 27 + 별도 사이트 API 8 + RAG 20 = 총 55개 데이터.** 매물 주소 → 좌표 → `HSMP_INNB`(단지고유번호 14자리)로 모든 데이터 join. 5장 카드 각자 핵심 데이터 + 보조 데이터 + 트리거 시 추가 데이터 매핑.

---

## 1. 마스터 키 구조

### 1.1. Primary Key — `HSMP_INNB`

한국부동산원 단지고유번호 14자리. **거의 모든 RAG 데이터(20개)의 join 키.**

### 1.2. 보조 키

| 키 | 용도 | 길이 |
|---|---|---|
| `PNU` | 필지번호 (토지대장) | 19자리 |
| `LNNO_ADRES` | 지번주소 | 가변 |
| `법정동코드` | 행정구 단위 join | 10자리 |
| `시군구코드` | 광역 단위 | 5자리 |
| `좌표 (위경도)` | 거리·반경 검색 | EPSG:4326 |

### 1.3. join 흐름

```
사용자 입력 (매물 주소)
  ↓ juso.go.kr 주소 표준화 + vworld 지오코딩
좌표 (위경도) + 정규화 주소
  ↓ 한국부동산원 단지 연계정보 (39번)
HSMP_INNB + PNU + 법정동코드
  ↓ 5장 카드 각자 데이터 호출
카드별 데이터 패널 (단지·인근·행정동·격자 단위)
```

### 1.4. join 방식별 분류

| join 단위 | 데이터 | 매칭 방식 |
|---|---|---|
| **단지 (HSMP_INNB)** | 39, 44~50, 52~58 (15건) | Primary key |
| **단지 (단지명+주소 fuzzy)** | 8, 9 공동주택 정보 | 단지명·주소 매칭 로직 필요 |
| **좌표 + 반경 검색** | 14·17·18·19·24·33·34 등 | 단지 좌표 ± Nm |
| **행정동 코드** | 20, 22 인구·세대 | 법정동 매칭 |
| **격자 (KATEC)** | 40, 41 데이터웨이 | 격자 ID 매칭 |
| **시군구 코드** | 25, 26 청약 / 35, 36 거시 | 5자리 매칭 |

## 2. 카드별 데이터 매핑

각 카드 = *별점 산출 핵심 산식* + *기여하는 데이터*.

### 2.1. Card 1 — 시세 분석

**별점 산출 원리**

> "이 호가가 적정한가?" — *회귀 잔차*와 *임대수익 환원율*로 검증. 잔차 작을수록 별점 ↑.

#### 핵심 데이터

| # | 데이터 | 별점 기여 방식 |
|---|---|---|
| 1 | 아파트 매매 실거래가 | 인근 단지 회귀 분석의 X·Y 데이터. 같은 행정동 + 비슷한 평형·연식 단지의 평당가 분포 → 회귀 적정가 추정 → 잔차 산출 |
| 2 | 아파트 전월세 실거래가 | 환산 월세 → 임대수익 환원율 = 연 임대수익 ÷ 매매가. 시장 평균과 가까우면 ★↑ |
| 3 | 연립·다세대 매매 실거래가 | 동일 지역 비교군 보완 (아파트 외 형식 매물 가격 분포) |
| 5 | 오피스텔 매매 실거래가 | 동일 지역 오피스텔 가격 → 입지 우월성 검증 |
| 52 | 단지별 직거래 정보 (한국부동산원 RAG) | 직거래 비율↑ = 증여·내부거래 의심 시그널. 평단가 + 전년 대비 변화율 |

#### 보조 데이터

| # | 데이터 | 활용 |
|---|---|---|
| 4 | 연립·다세대 전월세 | 빌라 매수 시나리오 |
| 6 | 단독·다가구 매매 | 단독 매수 시나리오 |
| 35 | ECOS 한국은행 | 거시 환경 보정 (금리 급등기 시장 전체 ★ 보정) |

#### 트리거 시 추가 데이터

- 잔차 ≥ 20% → 인근 단지 비교 그래프에 *직거래 비율*(52번) 추가 표시 → "정상 시세 왜곡 가능성" 시그널
