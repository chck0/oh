# VerifyHome — Information Architecture (SLC)

> 작성일: 2026-05-11
> 브랜치: feat/information-architecture
> 범위: P0 (랜딩) + P1 (시세 분석) — SLC

---

## 1. 서비스 개요

**VerifyHome** — 주소 하나로 아파트 시세를 데이터로 검증하는 서비스

---

## 2. 사이트맵

```
[P0] 랜딩 — page0-web.html
      주소 입력 → 검색
        ↓
[P1] 시세 분석 — market-web.html
      차트 + 챗봇
        ↓
      [PDF 저장] 대화 요약 다운로드
```

---

## 3. 페이지별 상세

### P0 — 랜딩 (`page0-web.html`)

| 항목 | 내용 |
|------|------|
| 역할 | 서비스 진입점. 주소 입력으로 분석 시작 |
| 레이아웃 | 단일 컬럼, 화면 중앙 정렬 |
| 핵심 컴포넌트 | 워드마크, 태그라인, 검색 바 |
| 입력 | 아파트 단지명 또는 도로명 주소 |
| 출력 | → P1으로 이동 (URL 파라미터로 주소값 전달) |
| 데이터 의존성 | 없음 (정적) |

---

### P1 — 시세 분석 (`market-web.html`)

| 항목 | 내용 |
|------|------|
| 역할 | 실거래가 차트 + AI 챗봇으로 시세 해석 |
| 레이아웃 | 2컬럼 (왼쪽 60% 차트 / 오른쪽 40% 챗봇) |

**컴포넌트 목록**

| 컴포넌트 | 설명 |
|----------|------|
| 상단 네비 | 로고 + 입력된 주소 표시 + "주소 다시 입력" (P0으로 이동) |
| 요약 스트립 | 시세 지표 3개 (6개월 상승률 등) |
| 차트 | Chart.js 라인차트 — 해당 아파트 시세 (같은 평수 기준) |
| 동 선택 패널 | 같은 구 내 다른 동 선택 → 차트에 선 추가 |
| 챗봇 | 항상 열려있음 — 차트 컨텍스트 기반 Claude API |
| PDF 저장 버튼 | 챗봇 대화 내용을 요약하여 텍스트 PDF로 다운로드 |

---

## 4. 데이터 흐름

```
P0 주소 입력
  → URL 파라미터 (?address=역삼래미안) 로 P1 전달
  → 실거래 API (LAWD_CD + DEAL_YMD)
  → 월별 평단가 계산 → ChartContext JSON
  → Chart.js 렌더링
  → 사용자 질문
  → FastAPI POST /api/chat (ChartContext + 대화 히스토리)
  → Claude API 응답
  → [PDF 저장 시] 대화 히스토리 → Claude API 요약 → jsPDF 다운로드
```
