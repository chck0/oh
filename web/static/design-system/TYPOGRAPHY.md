> SUIT 단일 폰트 시스템. 한글·영문·숫자 모두 동일 폰트.
> 부동산 서비스 특성상 **숫자 표현이 핵심**.

**버전**: 0.1
**최종 수정**: 2026-05-16
**관련 파일**: `typography.css`

## 1. 폰트 선택: SUIT

### 선택 이유
- iOS·안드로이드·웹에서 동일하게 렌더링 (디자인 일관성)
- **숫자 디자인이 우수** (San Francisco 스타일) — 부동산 = 가격·평수·통근시간 등 숫자 중심 서비스에 결정적
- SIL 오픈 폰트 라이선스로 상업적 무료
- 한글·영문·숫자가 한 폰트 안에서 자연스럽게 어우러짐 (Noto Sans+Inter 혼용 불필요)
- 토스·당근·우아한형제들 등 한국 주요 서비스의 사실상 표준

### 적용 방법
```html
<link rel="stylesheet" as="style" crossorigin
  href="https://cdn.jsdelivr.net/gh/sunn-us/SUIT/fonts/static/woff2/SUIT.css" />
```

또는 Next.js의 경우 로컬 폰트로 import (CDN 의존 제거):
```js
import localFont from 'next/font/local'
const suit = localFont({
  src: './fonts/SUITVariable.woff2',
  display: 'swap',
  weight: '45 920',
})
```

## 2. 핵심 원칙

### 2.1. 4개 웨이트만 사용
SUIT는 9개 굵기(100~900)를 제공하지만, **실제로는 4개만 사용**합니다.

| 웨이트 | 값 | 용도 |
|--------|---|------|
| Regular | 400 | 본문, 설명, 일반 텍스트 |
| Medium | 500 | 라벨, 강조, 단위(억/만/평/분) |
| Semibold | 600 | 제목, 가격, 메트릭 |
| Bold | 700 | 큰 타이틀, 가격 강조, 로고 |

**왜 4개만?** 더 많이 쓰면 위계가 모호해지고 일관성 깨짐. 300(Light)·800(ExtraBold) 등은 사용 금지.

### 2.2. Letter-spacing은 크기에 반비례
SUIT는 한글 가독성에 맞춰 영문/숫자가 약간 느슨한 편. 크기에 따라 자간 조정 필요.

| 크기 | letter-spacing |
|------|----------------|
| 36px+ (display) | -2% (`-0.02em`) |
| 28px (h1) | -1.5% (`-0.015em`) |
| 22px (h2) | -1% (`-0.01em`) |
| 18px (h3) | -0.5% (`-0.005em`) |
| 16px (body) | -0.25% (`-0.0025em`) |
| 14px 이하 | 0% |
| 11~12px overline | +10% (`+0.1em`) — 작은 글씨는 트래킹 넓혀 가독성 확보 |

### 2.3. 줄간격(line-height)도 크기별로 다르게
- 큰 타이틀(22px+): 1.2~1.3 (촘촘하게)
- 본문(16px): 1.6 (편안하게)
- 작은 라벨(11~12px): 1.4~1.5

## 4. 숫자 · 가격 표현 (가장 중요)

부동산 서비스의 정보 위계는 숫자에서 시작합니다. 이 규칙은 절대 흔들리지 않게 합니다.

### 4.1. 숫자와 단위 분리
가격·평수·시간 등 **숫자는 강하게, 단위는 가볍게**.

```html
<span class="num-bold">5</span><span class="unit">억</span>
<span class="num-bold">8,000</span><span class="unit">만</span>
```
- 숫자: 600 또는 700 weight, 큰 크기
- 단위: 500 weight, 숫자보다 한 단계 작은 크기 (16px 숫자 → 12px 단위)
- 색상: 숫자=primary(#1A1816), 단위=secondary(#4A4642)

### 4.2. 천 단위 콤마 항상
`5,800` ✅ / `5800` ❌

### 4.3. Tabular Numbers (자릿수 정렬)
리스트·표에서 숫자를 비교할 때는 **자릿수가 정렬**되도록.
```css
.num-aligned {
  font-feature-settings: 'tnum';
  /* 또는: font-variant-numeric: tabular-nums; */
}
```

매물 리스트, 가격 비교, 메트릭 카드 등에 필수 적용.

### 4.4. 맥락별 표기 분리

| 맥락 | 표기 | 이유 |
|------|------|------|
| 지도 핀 | `5.8억` | 공간 협소, 소수 1자리 |
| 매물 카드 (큰 가격) | `5억 8,000만` | 정확성 우선 |
| 검색 필터 | `5~6억` | 범위 표현 |
| 메트릭 (평수·시간) | `25평` / `27분` | 단위와 함께 |
| 비율·변동 | `+2.3%` / `-1.5%` | 부호 명시 |

### 4.5. 가격 타입 토큰

| 토큰 | 크기 | 웨이트 | 단위 크기 | 용도 |
|------|------|--------|-----------|------|
| `price-display` | 36px / 700 | 20px / 600 | 매물 상세 페이지 메인 가격 |
| `price-large`   | 28px / 700 | 16px / 500 | 매물 카드 가격 |
| `price-medium`  | 22px / 600 | 14px / 500 | 리스트 가격 |
| `price-small`   | 16px / 600 | 12px / 500 | 인라인 가격 |
| `price-pin`     | 13px / 600 | (포함됨)   | 지도 핀 (5.8억 줄임 표기) |

## 5. 한글-숫자 조합 시 주의

### 5.1. 한글 사이 숫자가 들어갈 때 띄어쓰기
- `25평 아파트` ✅
- `25평아파트` ❌

### 5.2. 단위 앞뒤 띄어쓰기 규칙
- 단위가 한국어일 때 (억/만/평/분/층): 붙여 씀 — `5억`, `27분`
- 단위가 영문/기호일 때: 띄어 씀 — `30 m²`, `100 km`

### 5.3. 큰 숫자 분리 위치
- 한국식: 4자리마다 (`1억 2,345만`)
- 콤마는 3자리마다 그대로 (`12,345`)
- 둘이 충돌할 때는 한국식 우선

## 6. 사용 예시 (실제 컴포넌트)

### 6.1. 매물 카드 헤더
```html
<h3 class="text-heading-3">서초동 래미안</h3>
<p class="text-body-sm text-secondary">서울시 서초구 서초동</p>
```

### 6.2. 매물 카드 가격
```html
<div class="price-large">
  <span class="num">5</span><span class="unit">억</span>
  <span class="num">8,000</span><span class="unit">만</span>
</div>
```

### 6.3. 메트릭 그룹
```html
<div class="metric">
  <span class="metric-label">통근</span>
  <span class="metric-value">27<span class="unit">분</span></span>
</div>
```

### 6.4. 베스트 매치 배지
```html
<span class="text-overline color-brand">BEST MATCH</span>
```

## 7. 접근성

- 본문 최소 크기 14px (모바일에서도)
- 줄간격 최소 1.5
- 가격 등 핵심 정보는 색상만으로 강조 금지 (굵기·크기 함께 사용)
- 작은 라벨(11px)은 반드시 letter-spacing 양수로 가독성 확보
