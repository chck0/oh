# 바둑이 아이콘 가이드

> 미니멀 노선. 데모 버전이며 가독성·정보 명료성이 최우선.
> "꼭 필요한 자리"만 정의하고 나머지는 텍스트로.

**버전**: 0.1
**최종 수정**: 2026-05-16
**관련 파일**: `icons.css`

---

## 1. 디자인 방향

바둑이는 **정보 중심 부동산 추천 서비스**입니다. 사용자가 가격·평수·통근시간을 빠르게 비교해야 하는 화면에 아이콘이 많으면 정보가 묻힙니다.

### 참고 노선
- **Linear** — 거의 텍스트만, 액션 아이콘 최소
- **Stripe** — 정보는 텍스트로, 액션만 아이콘
- **토스 초기** — 한글 라벨 우선, 아이콘 절제

### 안 따라갈 노선
- **네이버 부동산** — 매물 옵션마다 아이콘 (정보 노이즈 누적)
- **직방·다방** — 컬러풀 아이콘 가득 (캐주얼 톤이 목적)
- **일반 SaaS** — 모든 라벨 옆 아이콘 (장식적)

## 2. 5가지 사용 원칙

### 2.1. 텍스트로 충분하면 텍스트로
- ❌ `[♥ 저장]`
- ✅ `[저장]`

예외: 상단 툴바, 맵 컨트롤 등 **공간이 극히 좁은 곳**.

### 2.2. 한 화면에 아이콘 6개 이하
매물 카드, 검색바, 네비, 맵 컨트롤 모두 합쳐서. 이를 넘으면 무조건 어딘가 잘라냅니다.

### 2.3. 매물 옵션은 아이콘 안 씀 (중요)
부동산 앱의 흔한 함정. **매물 옵션은 모두 텍스트로** 처리합니다.

- ❌ `🛏 3  🛁 2  🚗 1`
- ✅ `침실 3 · 욕실 2 · 주차 1`

이유: 한글 라벨이 더 빨리 읽힙니다. 아이콘은 의미 해독 부담이 있어요.

### 2.4. Outline 1.5px 통일
- Filled 아이콘 금지 (`ti-heart-filled` 같은 거 사용 안 함)
- 두께 섞기 금지
- Tabler outline 기본값 그대로

예외: 활성화된 ♥ 만 filled. "내가 저장한 매물"임을 보여주는 유일한 fill.

### 2.5. 색은 회색이 기본
- 기본: `--color-text-secondary` (#4A4642)
- 호버: `--color-text-primary` (#1A1816)
- 빨강: 활성화된 ♥ 또는 ★ (베스트 매치)만
- 시맨틱 컬러: 알림 종류별 (성공/경고/정보)에만 한정

컬러 아이콘이 늘어날수록 정보 노이즈 증가. 절제 필수.

## 3. 라이브러리: Tabler Icons (Outline)

### 선택 이유
- **5,800개+** — 모든 케이스 커버
- **MIT 라이선스** — 상업적 무료
- **Outline 통일** — 미니멀 톤에 맞음
- **웹폰트 1.7KB** — 가벼움
- **stroke 1.5px 기본** — 손볼 필요 없음

### 적용
```html
<link rel="stylesheet"
  href="https://cdn.jsdelivr.net/npm/@tabler/[email protected]/dist/tabler-icons.min.css">

<!-- 사용 -->
<i class="ti ti-search"></i>
```

`size` 등 다른 클래스 사용 안 함. CSS `font-size`로만 조정.

---

## 4. 크기: 3단계만

| 크기 | 용도 |
|------|------|
| **16px** | 버튼·라벨 인라인, 작은 컨트롤 |
| **20px** | 검색바, 네비, 카드 헤더 |
| **24px** | FAB, 메인 액션, 큰 버튼 |

**금지**: 14px (너무 작아 안 보임), 18px·22px (어중간), 28px+ (장식적 과잉).

## 5. 실제 사용 아이콘 목록 (전부 합쳐 ~15개)

### 5.1. 네비게이션 & 검색 (6개)

| 용도 | 아이콘 | 위치 |
|------|--------|------|
| 검색 | `ti-search` | 검색바 좌측 |
| 필터 | `ti-adjustments-horizontal` | 검색바 우측, 필터 버튼 |
| 닫기 | `ti-x` | 모달·패널 우상단, 칩 제거 |
| 메뉴 | `ti-menu-2` | 모바일 햄버거 |
| 펼치기 | `ti-chevron-down` | 셀렉트, 아코디언 |
| 뒤로 | `ti-arrow-left` | 상세 페이지 헤더 |

### 5.2. 매물 액션 (2개)

| 용도 | 아이콘 | 위치 |
|------|--------|------|
| 저장 | `ti-heart` | 카드 우상단 (활성 시 빨강 fill) |
| 공유 | `ti-share-2` | 상세 페이지 |

### 5.3. 지도 컨트롤 (3개)

| 용도 | 아이콘 | 위치 |
|------|--------|------|
| 줌인 | `ti-plus` | 지도 좌하단 |
| 줌아웃 | `ti-minus` | 지도 좌하단 |
| 내 위치 | `ti-current-location` | 지도 좌하단 |

### 5.4. 상태 표시 (4개)

| 용도 | 아이콘 | 색상 |
|------|--------|------|
| 베스트 매치 | `ti-star` | `--color-brand-primary` (빨강) |
| 선택됨 | `ti-check` | `--color-status-success-text` (초록) |
| 안내/도움말 | `ti-info-circle` | `--color-status-info-text` (블루) |
| 주의 | `ti-alert-circle` | `--color-status-warning-text` (앰버) |

## 6. 아이콘 안 쓸 자리 (중요)

다음 자리에는 아이콘을 절대 추가하지 않습니다.

| 자리 | 대안 |
|------|------|
| 매물 옵션 (침실·욕실·주차·연식 등) | 텍스트만 (`침실 3 · 욕실 2`) |
| 상태 태그 내부 (`통근 27분`) | 색상으로 의미 구분 |
| 본문 단락 인라인 | 텍스트만 |
| 가격 옆 단위 (억·만·평·분) | 폰트 위계로 처리 (TYPOGRAPHY.md §4) |
| 푸터 링크 | 텍스트만 |
| 폼 라벨 옆 (`주소`, `평수`) | 라벨로 충분 |

### Before / After

**❌ 흔한 부동산 앱 스타일**
```
🏠 서초동 래미안
📍 서울시 서초구
💰 5억 8천만
🛏 3  🛁 2  🚗 1  📅 2018
```

**✅ 바둑이 스타일**
```
서초동 래미안
서울시 서초구
5억 8,000만
침실 3 · 욕실 2 · 주차 1 · 2018년
```

후자가 더 빠르게 읽히고 더 고급스러워 보입니다.

## 7. 사용 패턴

### 7.1. 아이콘만 (Icon-only) 버튼
공간 협소한 곳에서만. 반드시 `aria-label` 필수.

```html
<button class="btn btn-secondary btn-icon" aria-label="저장">
  <i class="ti ti-heart" style="font-size:18px;" aria-hidden="true"></i>
</button>
```

### 7.2. 텍스트 + 아이콘
액션이 명확한 메인 버튼.

```html
<button class="btn btn-primary btn-md">
  <i class="ti ti-search" style="font-size:16px;" aria-hidden="true"></i>
  매물 검색
</button>
```

아이콘이 텍스트 왼쪽, gap 6px.

### 7.3. 상태 표시
의미가 있는 아이콘. `aria-label`로 의미 보강.

```html
<span aria-label="베스트 매치">
  <i class="ti ti-star" style="color: var(--color-brand-primary);" aria-hidden="true"></i>
</span>
```

---

## 8. 접근성

| 상황 | 처리 |
|------|------|
| 아이콘만 단독 사용 | `aria-label="..."` 필수 |
| 텍스트와 함께 (장식) | `aria-hidden="true"` 명시 |
| 상태/의미 전달 | 부모 요소에 `role="img"` + `aria-label` |
| 색상만으로 의미 전달 금지 | 항상 텍스트 라벨 또는 추가 표시 병행 |

## 9. 의사 결정 체크리스트

새 아이콘을 추가할 때마다 다음을 자문:

1. **텍스트로 대체 가능한가?** → 예 → 텍스트 사용
2. **이 화면에 이미 아이콘 6개 있나?** → 예 → 추가 안 함, 다른 거 빼기
3. **매물 옵션 표현인가?** → 예 → 절대 추가 안 함
4. **outline 1.5px 스타일인가?** → 아니오 → 사용 안 함
5. **이미 정의된 ~15개 안에 있나?** → 아니오 → 추가 전 진짜 필요한지 재검토

다섯 개 모두 통과해야 사용.

## 10. 향후 확장 시 주의

서비스가 커지면 아이콘이 늘어나는 압력이 생깁니다. 그때 따를 규칙:

1. **새 아이콘 추가 시 이 문서에 반드시 추가** — 무서명 사용 금지
2. **유사 아이콘 통합** — `ti-x`, `ti-x-circle`, `ti-cancel` 중 하나만
3. **분기마다 사용 빈도 검토** — 한 번도 안 쓰는 아이콘은 가이드에서 제거
4. **시즌성 아이콘 금지** — 크리스마스 트리, 새해 폭죽 등
