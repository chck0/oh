# 바둑이 디자인 시스템

> 부동산 추천 서비스 "바둑이"의 컬러 시스템 가이드.
> 이 문서는 Claude·디자이너·개발자가 함께 읽는 단일 출처(single source of truth)입니다.

**버전**: 0.2 · Map Overlay Edition
**최종 수정**: 2026-05-16
**관련 파일**: `tokens.json`, `tokens.css`

---

## 1. 디자인 컨텍스트

### 서비스 성격
바둑이는 사용자의 **직장 주소를 기준으로 통근시간·평형·가격을 최적화**해서 베스트 부동산 매물을 추천하는 서비스입니다. 메인 화면은 지도 기반이며, 그 위에 검색·필터·매물 카드·핀 등의 UI가 얹혀집니다.

### 디자인 제약
- **지도가 화면의 70~90%를 차지함** — 지도 자체가 이미 색이 많은 캔버스
- **UI는 지도 위에 "떠 있는" 흰 패널 구조** — 네이버부동산, 호갱노노, 직방과 유사한 구조
- **모바일·데스크톱 동시 지원**
- **장시간 사용 시 눈에 편안해야 함** — 매물 탐색은 오래 걸리는 작업

### 톤 & 매너
- 친근하고 신뢰감 있는 한국형 서비스 톤 (당근, 토스, 오늘의집 계열)
- 사짜 부동산 느낌 ❌ / 프리미엄 럭셔리 ❌ / 친구 같은 동반자 ⭕
- 빨강 = 액션·강조의 시그니처 컬러 (브랜드 캐릭터 "바둑이"와 연결)

## 2. 핵심 원칙

### 2.1. 흰색이 베이스다
**모든 UI 패널의 배경은 순백(#FFFFFF)입니다.** 크림톤, 회색톤 베이스는 사용하지 않습니다. 이유:
- 지도 위에 떠 있는 패널은 흰색일 때 가장 분명하게 분리됨
- 컬러풀한 지도 캔버스와 충돌 최소화
- 가독성 최우선

### 2.2. 60·30·10 비율보다 95·4·1
일반적인 60·30·10 디자인 비율은 적용하지 않습니다. 이 서비스의 실제 면적 비율:

| 영역 | 비율 | 색상 |
|------|------|------|
| 지도 캔버스 | ~70% | (지도 자체 색상) |
| 흰색 UI 패널 | ~25% | `#FFFFFF` |
| 텍스트·아이콘·테두리 | ~4% | 그레이 스케일 |
| 빨강 액센트 | **~1% 이하** | `#A82E1B` |

**빨강을 5% 이상 쓰면 안 됩니다.** 빨강이 정보가 되려면 희소해야 합니다.

### 2.3. 그레이가 진짜 메인 컬러
화면의 대부분은 흰 배경 + 회색 텍스트·테두리로 구성됩니다. 그래서 회색 톤의 질이 시스템 전체의 인상을 결정합니다.

**중요**: 차가운 회색(#F5F5F5 계열) 대신 **2% 정도 옐로우-브라운이 섞인 따뜻한 회색**을 씁니다. 이유:
- 네이버지도·카카오맵의 베이지 톤 도로/녹지와 자연스럽게 어우러짐
- 부동산이라는 정서적 영역(집·거주)과 잘 맞음
- 차가운 기술 서비스 톤을 피함

### 2.4. 빨강은 액션과 추천에만
빨강(#A82E1B)을 쓰는 곳:
- ✅ 메인 CTA 버튼 (자세히 보기, 검색, 저장)
- ✅ 로고
- ✅ "베스트 매치" 추천 매물 핀
- ✅ 활성 필터 카운트 배지

빨강을 쓰지 않는 곳:
- ❌ 일반 매물 핀 (회색·흰색 사용)
- ❌ 본문 텍스트
- ❌ 일반 아이콘
- ❌ 비활성 상태
- ❌ 헤더 배경

### 2.5. 그림자가 깊이를 만든다
지도 위 패널이 "떠있는" 느낌의 핵심. 4단계 그림자로 위계를 만듭니다.

## 3. 컬러 토큰 (요약)

### 3.1. Primary — Baduki Red
부동산 추천이라는 액션과 강조에 사용. 벽돌 톤 빨강.

- **메인**: `--color-red-600` = `#A82E1B` (흰색 위 대비 7.4:1, WCAG AA 통과)
- **호버**: `--color-red-800` = `#7E2113`
- **활성/클릭**: `--color-red-900` = `#4D140C`
- **비활성**: `--color-red-300` = `#E89789`
- **배경 옅음**: `--color-red-50` = `#FDF4F2` (배지·태그 배경)

### 3.2. Neutral — Warm Gray
시스템의 진짜 베이스. 따뜻한 회색 스케일.

- **패널 배경**: `--color-white` = `#FFFFFF`
- **호버/섹션 구분**: `--color-gray-50` = `#FAF9F7`
- **카드 내부 구분**: `--color-gray-100` = `#F1EFEB`
- **테두리**: `--color-gray-200` = `#E8E6E3`
- **비활성 텍스트**: `--color-gray-500` = `#8A857E`
- **보조 텍스트**: `--color-gray-700` = `#4A4642`
- **메인 텍스트**: `--color-gray-900` = `#1A1816`

### 3.3. Semantic — 상태 색
모두 채도를 한 단계 낮춰 지도 위 마커와 충돌 방지.

| 의미 | 배경 | 텍스트/아이콘 | 사용처 |
|------|------|---------------|--------|
| Success | `#E6F0E5` | `#2D6A3F` | 시세 적정, 가격 하락, 매치 확인 |
| Warning | `#FBF1DC` | `#8C5E0E` | 시세 대비 높음, 매물 오래됨 |
| Info | `#E4ECF4` | `#2E4E6E` | 통근 시간, 거리 표시 |
| Brand | `#FDF4F2` | `#A82E1B` | 베스트 매치, 추천 |

## 4. 지도 핀 체계

부동산 서비스의 핵심 정보 위계는 지도 위 핀에서 나타납니다. 5종류 핀으로 정보 우선순위를 표현:

| 핀 종류 | 외형 | 의미 |
|---------|------|------|
| **추천 핀** | 빨강 배경 + 흰 텍스트 + 빨강 그림자 | 베스트 매치 (전체의 10% 이하) |
| **일반 핀** | 흰 배경 + 회색 외곽 + 검정 텍스트 | 표준 매물 |
| **본 매물 핀** | 연회색 배경 + 회색 텍스트 | 사용자가 이미 클릭한 매물 |
| **클러스터** | 흰 원 + 빨강 외곽 + 숫자 | 줌아웃 시 매물 그룹 |
| **선택된 핀** | 검정 배경 + 흰 텍스트 + 빨강 글로우 | 현재 상세 보기 중인 매물 |

**규칙**: 추천 핀은 화면 내 전체 핀 수의 10%를 넘지 않습니다. 모두가 추천이면 아무도 추천이 아닙니다.

## 5. 그림자 시스템

| 토큰 | 값 | 사용처 |
|------|---|--------|
| `--shadow-sm` | `0 1px 2px rgba(0,0,0,0.06)` | 툴팁, 작은 배지 |
| `--shadow-md` | `0 2px 8px rgba(0,0,0,0.08)` | 기본 핀, 카드 |
| `--shadow-lg` | `0 4px 16px rgba(0,0,0,0.10)` | 사이드 패널, 필터 패널 |
| `--shadow-xl` | `0 8px 28px rgba(0,0,0,0.14)` | 모달, 호버 시 카드 부각 |

**규칙**: 그림자는 검정이 아닌 검정 + 낮은 알파를 씁니다. 색이 있는 그림자(예: 빨강 그림자)는 추천 핀에만 예외적으로 허용.

## 6. 접근성

- **모든 텍스트·배경 조합은 WCAG AA 이상**: 본문 4.5:1, 큰 글씨 3:1
- **빨강만으로 정보 전달 금지**: 색맹 사용자를 위해 항상 텍스트 라벨 또는 아이콘 병행
- 예: "베스트 매치" 빨강 핀에는 ★ 아이콘도 함께 표시

## 7. 작업할 때 Claude에게 줄 컨텍스트

Claude Code 또는 Claude로 컴포넌트를 만들 때, 다음 한 줄과 함께 이 디렉토리를 참조시키세요:

```
이 폴더의 DESIGN_SYSTEM.md를 읽고, tokens.css의 변수를 사용해서
[원하는 컴포넌트] 를 만들어줘.
```

그러면 Claude가:
1. 디자인 의도를 이해하고
2. 토큰 변수를 사용한 일관된 코드를 생성하며
3. 빨강 면적 같은 규칙을 알아서 지킵니다.
